# TeamTrace Frontend

TeamTrace 前端工程，基于 Vue 3 + Vite，当前承载学生端、教师端、管理员端三套界面。

## 快速开始

```sh
# 1) 复制环境变量模板
cp .env.example .env

# 2) 安装依赖并启动开发环境
npm install
npm run dev
```

生产构建：

```sh
npm run build
```

构建说明：

- 前端路由已采用页面级懒加载，学生端、教师端、管理员端页面不会全部打进首屏主包。
- PDF 导出相关的 `html2pdf.js` 仍只在报告导出时动态加载。
- 当前生产构建已消除 Vite 默认的大 chunk 警告和教师端无效动态导入警告。

联调注意：

- `.env.example` 与 `vite.config.js` 当前默认指向 `http://localhost:8082`
- `.env.example` 默认 `VITE_APP_RUNTIME=live`，表示前端默认只走真实接口
- `.env.cloud.example` 默认 `VITE_APP_RUNTIME=live`，适合云端后端可用时做纯真实联调
- 如果本地已有旧的 `.env` 或 `.env.local`，请确认其中没有残留旧端口配置
- 若后端运行在其他端口，可在 `.env` 或 `.env.local` 中调整 `VITE_API_PROXY_TARGET`

## 运行模式

当前前端支持三种运行模式：

- `live`：所有请求直连真实接口，接口失败会直接中断当前操作
- `hybrid`：优先请求真实接口；接口不可用时自动回退本地演示数据
- `mock`：所有请求都使用本地演示数据，不依赖后端服务

当前主链路联调默认使用 `live`。

## 当前联调状态

### 认证

- 认证入口已新增系统介绍主界面：进入 `/auth` 先展示 TeamTrace 能力介绍、系统链路和登录/注册入口，视觉上采用品牌蓝白配色的首页式 Hero 与产品视窗；桌面端首屏已压缩到单视口内，并补充背景扫光、连接线发光、产品窗口轻浮动和按钮高光；点击后进入原注册登录表单；邀请码非必填，填写并校验成功则按教师注册，留空默认注册学生
- 注册成功后统一返回登录页，用户需手动输入手机号和密码登录；登录页默认不自动填充账号/密码，仅在用户主动勾选后记住手机号，并支持“忘记密码请联系管理员”提示
- 登录态使用按角色隔离的多会话存储：每个角色独立存储在 `localStorage` 的 `teamtrace_session_{role}` 中，当前活跃角色记录在 `teamtrace_active_role`
- 支持教师和学生多角色同时登录：教师登录后切换到学生登录，两个角色的 token 互不覆盖，可在 Shell 顶部个人面板中一键切换角色

### 学生端

- 学生端桌面壳层已统一为侧边导航 + 顶部工作区布局
- 首页已收口为四个高频入口卡片与快捷功能区，不再展示 mock 提醒与占位提示；首页班级接口失败时显示独立重试错误态，不再伪装成“没有班级”，加入班级邀请码输入会自动规范为 6-8 位大写字母和数字
- 支持教师、学生、管理员多角色同时登录：每个浏览器标签页独立维护活跃角色（`sessionStorage`），会话数据共享存储在 `localStorage` 的 `teamtrace_session_{role}` 中
- 在不同标签页中打开同一前端地址，可分别登录不同角色，互不干扰
- 登录页在已有其他角色会话时显示"快速切换已登录角色"按钮，也可直接输入新账号登录
- 学生端"加入班级"链路已收顺为：首页快捷入口可直达邀请码弹窗；加入成功后班级中心会高亮刚加入的班级，并可立即进入班级
- 班级中心加载失败时显示独立重试错误态，不再把接口失败伪装成“没有班级”；筛选无结果时提供清空筛选入口
- 班级中心、班级详情、班级小组、任务中心已完成一轮学生视角重构；班级详情主接口失败时显示独立重试错误态，任务/小组辅助数据失败时保留班级基础信息并给出局部提示，不再把接口失败显示成“暂无任务”或“未加入小组”；班级详情只保留班级信息、班级小组、班级任务三个真实业务页签，不再展示当前后端未支持的同学名单占位页；小组进度改为进入“小组”页签后按需加载，失败时显示局部不可用状态，避免打开班级首屏就请求所有小组进度
- 学生首页、班级中心、任务中心、申诉中心现已对齐真实聚合接口：`/api/student/classes`、`/api/student/tasks`、`/api/student/appeals`
