# TeamTrace UX 架构方案 v1

> 基于 2026-05-12 实际代码审计 + PRD 对照 + 用户痛点的完整 UX 重构方案
> 
> **核心原则：基于现有代码演进，不推翻重来。** 所有改动都是增量修补，不是推倒重建。

---

## 0. 现状审计：当前 UX 的核心问题

经过对 49 个 Vue 文件、~100 个 API 端点、PRD、设计规范的全面审计，诊断出以下根因：

### 🔴 P0 — 阻塞用户体验的根本问题

| # | 问题 | 具体表现 | 影响 |
|---|------|---------|------|
| 1 | **页面没有"任务引导"** | 教师登录后看到 4 张统计卡片，但不知道接下来该干什么；学生看到作业列表但没有"先加入班级"的前置提示 | 新用户迷失，老用户效率低 |
| 2 | **大页面无信息层级** | StudentTaskDetailView(1930行)、TeacherDashboardView(1940行)、TeacherScoresCenterView(1933行) 把所有功能堆在一个长页面里，没有折叠/分组/优先级区分 | 用户找不到想要的功能 |
| 3 | **跨页面跳转断裂** | `/teacher/classes/:classId/tasks/:taskId/appeals` → 重定向到 `/teacher/appeals`；`/student/classes/:classId/tasks` → 重定向到 `/student/tasks`。用户点击"申诉"却跳到另一个完全不同的页面 | 上下文丢失，用户困惑 |
| 4 | **三端壳层不一致** | Admin 用 `AdminShell`（props 传入 title），Teacher 用 `TeacherShell`（内置路由+侧边栏+通知徽章），Student 用 `StudentShell`（问候语+侧边栏）。三套完全不同的交互模式 | 同一产品三套体验 |

### 🟠 P1 — 影响效率的交互问题

| # | 问题 | 具体表现 |
|---|------|---------|
| 5 | **工具栏/筛选区没有统一位置** | 有的页面筛选在顶部，有的在侧边，有的藏在弹窗里 |
| 6 | **空状态处理不一致** | 有的页面显示"暂无数据"，有的显示占位符，有的显示引导文案 |
| 7 | **操作反馈不明确** | 创建班级/发布任务后没有明确的"下一步"引导 |
| 8 | **页面太重** | 5 个页面超过 1500 行，一个 Vue 文件承担了 4-5 个职责 |

### 🟡 P2 — 视觉一致性问题

| # | 问题 |
|---|------|
| 9 | 统一布局规范（611行）已写好但未应用到现有页面 |
| 10 | 主题 CSS（admin/teacher/student）分散，Token 不统一 |
| 11 | 玻璃液态效果在不同页面表现为不同强度 |

### 1.1 教师完整使用路径

```
│   │   │                                                                 │
│   │   ├── 查看某个任务 → 任务详情                                      │
```

```
│  📋 今日待办                                  │
│  ┌──────────────────────────────────────┐    │
│  │ 🔴 软件工程A班 — 3件新申诉待处理       │ →  |    │
│  │ ⚠️ 数据结构B班 — 作业周五截止(6人未提交)│ →  |    │
│  │ ⚪ 操作系统C班 — 刚发布任务，等待学生加入 │ →  |    │
```

## 2. 信息架构重构（Information Architecture）

### 2.1 页面层级重新分组

**现有问题**：路由全部平铺在 Shell 下，没有体现"主要页面"和"辅助页面"的区别。

**改进方案**：三级信息架构——

```
教师端
├── 🏠 工作台（/teacher）← 登录后第一眼
│   ├── 待办摘要
│   ├── 班级快览
│   └── 通知预览
│
├── 📚 班级空间（核心工作区）
│   ├── 班级列表（/teacher/classes）
│   ├── 班级详情（/teacher/classes/:classId）
│   │   ├── [Tab] 概览（班级信息 + 任务列表 + 快捷操作）
│   │   ├── [Tab] 学生（/teacher/classes/:classId/students）
│   │   ├── [Tab] 分组（/teacher/classes/:classId/groups）
│   │   └── [Tab] 任务（/teacher/classes/:classId/tasks）
│   │
│   └── 任务详情（/teacher/classes/:classId/tasks/:taskId）
│       ├── [Tab] 概览（任务信息 + 小组进度概览）
│       ├── [Tab] 进度看板（/teacher/.../progress）
│       ├── [Tab] 互评（/teacher/.../peer-reviews）
│       ├── [Tab] 成绩（/teacher/.../score-summaries）
│       └── [Tab] 申诉（/teacher/.../appeals）← 不再跳转！
│
├── 📊 数据中心（独立入口）
│   ├── 评分中心（/teacher/scores）← 跨班级
│   ├── 申诉中心（/teacher/appeals）← 跨班级
│   ├── 报告中心（/teacher/reports）
│   └── 数据分析（/teacher/dashboard）
│
└── ⚙️ 个人
    ├── 通知与日志（/teacher/notifications-logs）
    ├── 帮助中心（/teacher/help）
    └── 个人中心（/teacher/profile）
```

**关键改动**：
1. **申诉不再跳转**：`/teacher/classes/:classId/tasks/:taskId/appeals` 现在是跳转页 → 改为**原地 Tab 展示**该任务的申诉
2. **班级详情改为 Tab 结构**：概览/学生/分组/任务放同一个班级上下文中，减少页面跳转
3. **任务详情改为 Tab 结构**：5 个 Tab 替代 5 个独立路由 + 1 个跳转页

```
学生端
```

**关于"弹窗 vs 独立页"的决策**：

| 操作 | 形式 | 理由 |
|------|------|------|
| 发布任务 | **独立页** | 字段多，需要看到班级上下文 |
| 创建班级 | **独立页** | 新用户引导，需要集中注意力 |
| 生成邀请码 | **弹窗** | 操作简单，1 个下拉 + 1 个按钮 |
| 创建小组 | **弹窗** | 操作简单，在分组管理页内完成 |
| 加入班级（学生） | **弹窗** | 输入邀请码即可，极简 |
| 提交子任务 | **弹窗** | 填写说明 + 上传附件 |
| 处理申诉 | **弹窗/抽屉** | 查看原文 + 填写处理意见 |

### 4.1 教师"发布任务"完整交互流

```
    ↓
```

```
任务详情页（Tab: 我的任务）

  ┌──────────────────────────┐

  │ 完成说明：                │
  │ ┌────────────────────┐   │
  │ │ 已完成XX文献的整理... │   │
  │ └────────────────────┘   │
  │                          │
  │ 附件：[选择文件]          │
  │                          │
  │      [取消]  [提交审批]   │
  └──────────────────────────┘
    ↓
提交成功 → 子任务状态变为"待审批"
    ↓
页面内反馈：子任务行高亮 + 状态徽标变化
```

```
任务详情页（Tab: 我的任务）

    ↓
```

## 8. 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 申诉：跳转 vs 原地 Tab | **原地 Tab** | 用户处理申诉需要看到任务上下文，跳转会丢失 |
| 任务详情：多个路由 vs 一个 Tab 页 | **一个 Tab 页** | 减少页面跳转，保持筛选状态（组选择器） |
| 班级详情：独立路由 vs Tab | **Tab** | 教师在班级内操作切换频繁，Tab 更快 |
| 发布任务：弹窗 vs 独立页 | **独立页** | 字段多（4-5个），弹窗太挤 |
| 创建小组：弹窗 vs 独立页 | **弹窗** | 操作简单，在分组管理页内完成 |
| 提交子任务：弹窗 vs 内联展开 | **弹窗** | 需上传附件，内联展开空间不够 |
