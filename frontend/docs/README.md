# TeamTrace Frontend Docs

本目录用于收纳前端仓内的非代码文档，避免继续散落在仓库根目录。

## 目录结构

- `status/frontend_data_source_matrix.md`
  - 页面级数据来源、真实接口接通状态、后端支持状态
  - 这是前端联调现状的主真值文档
- `unified-layout-rules-v1.md`
  - 当前三端统一布局规范草案
  - 用于约束管理员端优先落地、教师端保护线、学生端后续映射
- `archive/`
  - 阶段性开发归档、收尾总结、交接说明
- `reference/frontend_demo*.md`
  - 历史 demo、设计草案、提示词和页面参考
  - 仅作参考，不作为当前实现真值
- `reference/DESIGN-apple.md`
  - Apple 风格视觉母版参考
  - 当前已被吸收到 `unified-layout-rules-v1.md`，不直接作为实现真值
- `ux-architecture-v1.md`
  - **UX 架构方案** — 基于代码审计的完整用户体验重构方案
  - 包含：用户旅程地图、信息架构、页面布局模板、交互逻辑、实施优先级
- `ux-interaction-design-v1.md`
  - **屏幕级交互设计** — 每个页面状态、每个按钮的跳转逻辑、每个操作的连锁反应
  - 包含：教师/学生/管理员全场景、空状态引导、操作后反馈、全局交互规则
- `ux-edge-cases-v1.md`
  - **边界情况 + 细节交互设计** — 每个模块在异常/边界/并发/空数据下的行为
  - 包含：11个模块的细节、全局加载/空/错误状态、浏览器行为、移动端注意事项
- `ux-gap-analysis-v1.md`
  - **遗漏分析（产品经理视角）** — 以"一个学期完整使用"为时间轴的逐帧审查
  - 包含：用户生命周期、并发冲突、时间维度、情感设计、移动端、10项遗漏清单
- `teacher-class-setup-flow-spec-v1.md`
  - **教师端班级建成链交互规格** — 教师工作台空状态、创建班级、邀请码、等待学生、班级详情与学生/分组/任务入口的落地规格
  - 当前作为教师端页面重构的施工规格，工作台、班级中心、班级详情首屏、学生管理、分组管理和班级任务页已按第一轮规格落地；班级内页面与教师任务详情已进一步按“一个页面一个主工作区”去除重复统计卡与冗余层级，学生管理页已补大班级名单概况、当前页分组状态、下一步入口和学生接口失败独立错误态；分组人数规则已改为至少 1 人即可成组，不再固定 2-5；分组创建弹窗已改为适合大班级操作的搜索、筛选、已选成员和组长设置工作区；班级任务页与跨班级任务中心的发布表单已收口到真实接口字段并补发布成功高亮反馈；任务详情进度页已改为按需加载，并二次重设计为小组处理队列与当前小组处理台；数据分析页已改为顶部命令栏、左侧信号栏、中间处理列表和右侧详情的同屏工作台，桌面端三栏固定在同一工作区，中间处理列表采用固定标题和内部滚动，左侧当前范围压缩为轻量上下文行，个人维度使用紧凑行式列表承载大班级数据，评分维度仍按需加载成绩聚合
- `student-redesign-plan.md`
  - 学生端重构归档入口，指向已完成的学生端收尾文档；当前继续迭代时以学生端真实业务页为准，首页、班级中心和班级详情已补加载失败重试态，班级中心已补筛选空态，班级任务入口已改为纯路由桥接到任务中心，班级详情只保留真实可用的班级信息、班级小组、班级任务三个页签并移除无学生名单接口支撑的同学占位页，小组页签已改为按需加载进度，小组中心已收口为小组身份、成员、邀请码、任务入口和审批的协作工作区，并补齐组长移除普通组员入口与二次确认，互评工作台已补齐学生提交互评入口与匿名记录查看，成绩汇总页已拆分身份加载失败、成绩接口失败、未入组、待审批和暂无成绩状态，任务申诉页已兼容真实接口 `id` 与 hybrid 回退 `appealId`，申诉中心已改为状态队列和详情同屏工作区，通知中心已改为通知队列和详情同屏工作区，个人中心已改为资料概览、编辑资料、安全操作和最近操作记录工作区，学生操作记录已接通真实日志查询与导出接口；任务详情、班级小组、班级详情、任务中心、申诉中心、通知中心和互评工作台已统一为“主数据失败显示可重试错误态，辅助数据失败保留主工作区并局部提示”的前端状态策略

## 维护原则

- 页面是否已接通真实接口，以 `status/frontend_data_source_matrix.md` 为准
- 阶段性收尾、冻结、归档类文档统一进入 `archive/`
- 历史设计稿、提示词、参考草案统一进入 `reference/`
- 前端整体事实说明、启动方式、联调入口，以仓库根目录 `README.md` 为准
- 样式 token 模板继续放在 `src/styles/*.md`，因为它们直接服务于样式源码
