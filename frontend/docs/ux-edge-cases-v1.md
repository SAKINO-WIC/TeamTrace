# TeamTrace 细节交互 + 边界情况设计 v1

> **本文档回答的问题是：每个功能在"非理想情况"下应该怎么表现。**
> 
> 一个系统不是靠主流程好用的，是靠边界情况不翻车好用的。

---

# 模块 1：注册 / 登录

### 1.1 邀请码输入

| 情况 | 行为 |
|------|------|
| 学生注册不填邀请码 | 正常注册为学生 |
| 教师注册填了邀请码 | 校验邀请码 → 通过则注册为教师 |
| 邀请码已过期 | **显示"此邀请码已过期（有效期30天），请联系管理员重新获取"** |
| 邀请码已被使用 | **显示"此邀请码已被使用，每个邀请码只能注册一个教师账号。请联系管理员获取新邀请码"** |
| 邀请码已被撤销 | **显示"此邀请码已被管理员撤销，请联系管理员"** |
| 邀请码格式不对 | **实时校验：输入框下方提示"邀请码应为6-8位字母数字组合"** |
| 输入过程中 | 输入框实时转大写（邀请码不区分大小写，但统一显示大写避免混淆） |

### 1.2 手机号输入

| 情况 | 行为 |
|------|------|
| 输入非数字 | 自动过滤，只允许输入数字 |
| 不足11位时点注册 | 按钮保持 disabled，输入框下提示"请输入11位手机号" |
| 已注册的手机号 | 提交后返回错误 → 显示"该手机号已注册，请直接登录。 [去登录]" |
| 手机号格式校验 | 1开头的11位数字，输入满11位时自动触发格式校验 |

### 1.3 密码输入

| 情况 | 行为 |
|------|------|
| 输入过程中 | 实时显示密码强度（弱/中/强）+ 格式要求 |
| 不足8位 | 提示"密码至少需要8位" |
| 纯数字或纯字母 | 提示"密码需要同时包含字母和数字" |
| 超过20位 | 限制输入，最多20位 |
| 显示/隐藏密码 | 眼睛图标切换明文/密文，默认密文 |
| 注册时两次密码不一致 | 注册页需要"确认密码"字段，不一致时提示"两次输入的密码不一致" |

### 1.4 登录

| 情况 | 行为 |
|------|------|
| 手机号不存在 | 显示"该手机号未注册，请先注册。 [去注册]" |
| 密码错误 | 显示"密码错误，请重试" + **连续输错 5 次后锁定 15 分钟**（或显示"忘记密码？"引导） |
| 账号被禁用 | 显示"你的账号已被管理员禁用，请联系管理员" |
| 登录过程中 | 按钮显示 loading 转圈 + 文字变为"登录中..."，按钮 disabled |
| 网络超时 | 显示"网络连接超时，请检查网络后重试。 [重试]" |
| 记住手机号 | 勾选后下次打开登录页自动填入手机号（但不清空密码） |
| 多角色已登录 | 如果已有其他角色登录，登录页顶部显示"你已登录为[教师/学生]，[切换角色]" |

### 1.5 忘记密码

| 情况 | 行为 |
|------|------|
| 当前实现 | ⚠️ PRD FR-02 要求但未实现，登录页显示"忘记密码？请联系管理员"即可 |
| 后续实现 | 验证码 → 新密码 → 确认密码 → 重置 |

# 模块 2：班级管理

### 2.1 创建班级

| 情况 | 行为 |
|------|------|
| 班级名称为空 | 按钮 disabled，输入框下提示"请输入班级名称" |
| 班级名称过长 | 限制 50 字，超出时不允许继续输入 |
| 小组人数 | 不再要求固定范围；创建小组只校验至少选择 1 名成员 |
| 学期选择 | 默认为当前学期（自动识别 3-8月=春季，9-2月=秋季） |
| 创建中 | 按钮显示 loading + "创建中..."，按钮 disabled |
| 创建失败 | 显示具体错误原因（如"班级名称已存在"？如果后端不校验则显示通用错误） |

### 2.2 班级邀请码

| 情况 | 行为 |
|------|------|
| 复制邀请码 | 点击复制后按钮文字变为"✅ 已复制"，2 秒后恢复 |
| 刷新邀请码 | **弹窗确认："刷新后旧邀请码将立即失效，确定？"** [取消] [确认刷新] |
| 邀请码已过期 | 显示红色"已过期"标签 + 自动出现"生成新邀请码"按钮 |
| 邀请码已被使用 | 显示灰色"已使用"标签（已被某个学生使用） |
| 未生成邀请码 | 显示"暂未生成邀请码 [生成]" |
| 刷新生效中 | 按钮 loading + "生成中..." |

### 2.3 解散班级

| 情况 | 行为 |
|------|------|
| 点击解散 | **弹窗确认：**<br>"确定解散「软件工程A班」吗？<br>解散后：<br>• 所有任务和数据暂时保留（30天内可恢复）<br>• 学生将无法访问此班级<br>• 30天后数据将永久删除"<br>[取消] [确认解散] |
| 解散中 | 按钮 loading + "解散中..." |
| 解散成功 | 从班级列表移除，顶部显示提示条"班级已解散，30天内可恢复。[撤销]" |
| 撤销解散 | 点击[撤销] → 班级恢复，学生可重新访问 |
| 班级有未完成任务 | 解散时额外提示"该班级有 3 个进行中的任务，解散后学生将无法继续完成" |

### 2.4 学生管理

| 情况 | 行为 |
|------|------|
| 移除学生 | **弹窗确认："确定将[张三]移出班级？移除后该学生的历史数据保留但无法参与新任务。"** [取消] [确认移除] |
| 移除后 | 学生从列表消失（动画淡出），班级人数 -1，顶部提示"已将张三移出班级" |
| 学生列表为空 | 显示"暂无学生加入。邀请码：ABC123 [复制] 发给学生即可加入" |
| 学生搜索无结果 | 显示"未找到匹配的学生" + 清空搜索按钮 |
| 分页边界 | 删除当前页最后一条时 → 自动回到上一页（如果非首页） |

### 2.5 分组锁定

| 情况 | 行为 |
|------|------|
| 切换锁定 | 滑块切换即生效，无弹窗确认（操作可逆） |
| 锁定后 | 分组管理页所有"添加/移除/创建"按钮 disabled，hover 显示"分组已锁定" |
| 解锁后 | 按钮恢复可用 |

# 模块 3：分组管理

### 3.1 创建小组

| 情况 | 行为 |
|------|------|
| 小组名称为空 | disabled 按钮 + 提示 |
| 选择组长 | 下拉框列出所有已选成员 |
| 未选择任何成员 | 提示"请至少选择 1 名学生" |
| 学生已被其他组选中 | 已被选的学生在下拉框中置灰 + 显示"已在第X组" |
| 创建成功 | 弹窗关闭，分组列表刷新，新组卡片短暂高亮 |

### 3.2 移动成员

| 情况 | 行为 |
|------|------|
| 目标组已满（≥最大人数） | 该组在下拉选项中置灰 + 显示"已满 (5/5)" |
| 移动组长 | **弹窗确认："[张三]是当前组的组长，移动后该组将没有组长。确定？"** [取消] [移动] |
| 移动成功 | 两种方式：拖拽 / 下拉选择 → 即时生效 |
| 拖拽到无效区域 | 动画回弹到原位 |
| 移动失败 | 显示具体错误（如"该学生已不属于此班级"） |

### 3.3 学生自建小组（学生端）

| 情况 | 行为 |
|------|------|
| 班级分组已锁定 | "创建小组"按钮 disabled，hover 显示"班级分组已被教师锁定" |
| 学生已在其他组 | "创建小组"按钮隐藏或 disabled，提示"你已在第X组" |
| 创建时设置加入模式 | 直通 / 审批 — 单选按钮，默认直通 |
| 邀请码有效期 | 默认 24 小时，下拉可选（1h/6h/12h/24h/48h/7天） |
| 组长刷新邀请码 | 旧码立即失效，新码生成 |

### 3.4 加入小组审批

| 情况 | 行为 |
|------|------|
| 有新的入组申请 | 组长侧边栏或小组页出现红色徽标数字 |
| 审批通过 | 申请记录状态变为"已通过"，成员列表刷新 |
| 审批拒绝 | 申请记录状态变为"已拒绝"（申请人不收到通知？→ 需要，显示"入组申请被拒绝"） |
| 组长移除成员 | **弹窗确认："确定将[李四]移出小组？"** [取消] [移除] |

# 模块 4：任务管理（教师端）

### 4.1 发布任务

| 情况 | 行为 |
|------|------|
| 任务名称为空 | disabled 按钮 + 提示 |
| 截止时间早于当前时间 | 实时校验提示"截止时间不能早于当前时间" |
| 开启互评后未设时长 | 互评时长为必填，默认 24 小时 |
| 互评时长输入 | 只允许正整数，范围 1-168（1小时到7天） |
| 任务描述过长 | 限制 2000 字，超出不允许输入 |
| 发布中 | 按钮 loading + "发布中..." |
| 发布成功 | 回到班级详情 + 任务列表刷新 + 新任务行高亮 + 顶部提示"任务已发布" |

### 4.2 编辑任务

| 情况 | 行为 |
|------|------|
| 任务已有学生提交 | 编辑时提示"该任务已有学生提交，修改截止时间或描述可能影响学生" |
| 修改截止时间 | 只能延后不能提前（或提前时弹窗确认） |
| 修改互评设置 | 如果互评已开始 → 不可关闭互评，只能延长时间 |
| 删除任务 | **弹窗确认："确定删除「文献综述报告」？删除后所有子任务、互评、成绩将被一并删除。"** [取消] [确认删除] |

### 4.3 教师任务详情（Tab 页）

**组选择器的边界情况**：

| 情况 | 行为 |
|------|------|
| 只有 1 个组 | 自动选中该组，选择器隐藏（不需要选） |
| 没有组 | 显示"该班级暂无分组，请先 [管理分组]" |
| 选择器切换组 | 当前 Tab 内容自动刷新，显示 loading |
| 刷新失败 | 显示"加载失败 [重试]" |

**各 Tab 的边界情况**：

| Tab | 边界情况 | 行为 |
|-----|---------|------|
| 进度 | 组员未认领任何子任务 | 显示"—" |
| 进度 | 子任务刚被组长删除 | 刷新后该行消失，不影响其他数据 |
| 互评 | 该组无人提交互评 | 显示"该组暂无互评记录" |
| 互评 | 互评未开启 | 显示"该任务未开启互评" |
| 成绩 | 教师未打分 | 教师评分为空，综合成绩为空，显示"待评分" |
| 成绩 | 互评为空+教师未打分 | 全部显示"—"，综合成绩显示"暂无数据" |
| 申诉 | 无申诉 | 申诉 Tab 的徽标为 0 或隐藏数字 |
| 申诉 | 已处理的申诉 | 默认不显示（筛选"待处理"），可通过"全部"查看 |

# 模块 5：子任务工作流（学生端）

这是最复杂的交互模块，涉及 6 个状态转换。

### 5.1 子任务状态机

```
待认领 ──→ 进行中 ──→ 待审批 ──→ 已完成
  ↑          ↑          │
  │          │          ├──→ 进行中（打回）
  │          │          │
  │          └──────────┘
  │
  └── 组长可以重新指派
```

每个状态的颜色和图标：

| 状态 | 颜色 | 图标 | 可执行操作 |
|------|------|------|----------|
| 待认领 | 灰色 | ○ | [认领]（任何组员） |
| 进行中 | 蓝色 | ◉ | [提交]（负责人）/ [重新指派]（组长） |
| 待审批 | 橙色 | ◑ | [通过] [打回]（组长） |
| 已完成 | 绿色 | ● | — |
| 已逾期 | 红色 | ⚠ | [提交]（仍可提交） |

### 5.2 认领子任务

| 情况 | 行为 |
|------|------|
| 同时多人认领 | 后端应使用乐观锁（已有 `@Version`），先到先得 → 失败者看到"该子任务已被其他人认领" + 列表自动刷新 |
| 学生已认领多个 | 没有限制（PRD 无限制），但每组每个任务的所有子任务是组内共享的 |
| 学生不在该组 | 按钮不显示（权限控制在前端路由 + 后端接口层） |

### 5.3 提交子任务

| 情况 | 行为 |
|------|------|
| 输入为空 | disabled 按钮 + 提示"请填写完成说明" |
| 附件上传 | 支持格式：jpg/png/pdf/docx/xlsx/zip，大小限制 10MB |
| 附件超大 | 上传时拦截，提示"文件大小不能超过 10MB" |
| 上传中 | 进度条 + 百分比 |
| 上传失败 | 提示"上传失败 [重试]" |
| 提交中 | 按钮 loading + "提交中..." |
| 提交成功 | 弹窗关闭，子任务状态变为"待审批"
| **提交草稿** | 如果学生填写了内容但没提交就关闭弹窗 → **弹窗确认："你有未提交的内容，确定关闭？" [继续编辑] [放弃]** |
| 子任务已被删除 | 提交时后端返回错误 → 提示"该子任务已被删除" + 列表刷新 |
| 子任务已被重新指派 | 提交时后端返回错误 → 提示"该子任务已被重新指派给其他人" + 列表刷新 |

### 5.4 组长审批子任务

| 情况 | 行为 |
|------|------|
| 通过 | 弹窗确认或直接操作（一步完成，不需要再弹确认） → 状态变为已完成 |
| 打回 | 展开输入框填写打回原因（必填） → 状态变为进行中 |
| 打回原因为空 | disabled 按钮 + 提示"请填写打回原因" |
| 审批时子任务已被删除 | 提示"该子任务已被删除" |
| 连续审批 | 审批完一个后自动展示下一个待审批（如果有） |

### 5.5 组长拆解子任务

| 情况 | 行为 |
|------|------|
| 子任务截止时间晚于父任务 | 实时校验，提示"子任务截止时间不能晚于总任务截止时间" |
| 指定负责人 | 下拉框列出该组所有成员（不包括已退出的） |
| 修改已发布的子任务 | 修改截止时间时，如果新截止时间早于原截止时间并且负责人还没提交 → 发送通知提醒 |
| 删除子任务 | **弹窗确认："确定删除「引言撰写」？如果已有提交内容，将一并删除。"** [取消] [删除] |

# 模块 6：互评

### 6.1 互评时间窗口

| 情况 | 行为 |
|------|------|
| 互评未开启 | 互评 Tab 显示倒计时："互评将在任务截止后开启" |
| 互评进行中 | 显示剩余时间倒计时（精确到秒） |
| 互评已截止 | 显示"互评已关闭"，不可再评分 |
| 教师在截止后延长时间 | 互评重新开放，倒计时更新，学生收到通知 |
| **截止前最后一秒提交** | 客户端时间 ≤ 截止时间即可提交（后端也做校验，双保险） |
| **倒计时归零瞬间** | 页面内如果有未提交的评分 → 弹窗提示"互评已截止，你的评分未提交" + 自动关闭 |

### 6.2 匿名互评

| 情况 | 行为 |
|------|------|
| 匿名代号 | 每个学生在该任务中拥有唯一代号（如"蓝色海豚"），全员可见 |
| 组成员只有 1 人 | 没有可评对象，显示"你的小组暂无其他成员" |
| 组员 = 2 人 | 只需评 1 个人 |
| 评分范围 | 1-100，整数。输入非数字字符时自动过滤。如果输入 >100 或 <1，提示"评分范围 1-100" |
| 评分备注 | 可选，限制 200 字 |
| 提交前 | **弹窗确认："提交后不可修改，确定？"** [再检查一下] [确认提交] |
| 提交后 | 该组员行变为"已评 ✓"，不可再编辑 |
| 教师查看互评 | 教师看不到匿名代号→真实姓名的映射（设计如此，保护匿名性） |

### 6.3 互评数据一致性

| 情况 | 行为 |
|------|------|
| 互评过程中被踢出组 | 前端实时检查：如果不再属于该组 → 互评页面显示"你已不在该小组"，不可提交 |
| 组员被移除后互评怎么算 | 已提交的互评保留，未提交的不需要 |
| 新的组员加入 | 新组员的互评列表自动包含新成员（如果是"相互评"模式） |

# 模块 7：评分与成绩

### 7.1 教师打分

| 情况 | 行为 |
|------|------|
| 输入非数字 | 自动过滤 |
| 输入分数范围 | 1-100，超出提示 |
| 未打分的显示 | "—" 或 "待评分" |
| 打分保存 | **即时保存**（失去焦点或按回车时自动保存，不需要点"保存"按钮）。保存时该行短暂显示"✅" |
| 保存失败 | 显示"保存失败 [重试]"，分数恢复原值 |
| 修改分数 | 直接修改后失焦即保存 |
| 全部打分完成 | 顶部出现提示条"所有学生已打分完成，[通知学生成绩已发布？]" |
| 学生成绩为纯 0 | 如果学生没有互评也没有教师评分 → 综合成绩显示"—"（不是 0，因为 0 会被认为是有分数） |

### 7.2 成绩发布

| 情况 | 行为 |
|------|------|
| 发布前 | **弹窗确认："发布后所有学生将立即看到成绩，且可在申诉期内提交申诉。确定发布？"** [取消] [发布成绩] |
| 发布后 | 学生端成绩 Tab 解锁 + 全员收到通知 |
| 发布后修改 | 教师仍可修改分数（因为申诉可能导致改分），修改后成绩实时更新 |
| 申诉中的成绩 | 如果分数因申诉被调整 → 成绩旁边显示"已调整"标签 + 原分数（划线）+ 新分数 |

### 7.3 课程总评

| 情况 | 行为 |
|------|------|
| 自动计算 vs 手动输入 | 系统自动汇总（所有任务加权），教师可手动覆盖 |
| 手动覆盖后 | 显示"已手动调整"标签，旁边有"恢复自动计算"按钮 |
| 一个任务都没完成 | 课程总评显示"暂无数据" |

# 模块 8：申诉

### 8.1 学生发起申诉

| 情况 | 行为 |
|------|------|
| 成绩未发布 | 申诉入口隐藏或 disabled，提示"成绩发布后可提交申诉" |
| 同一事项重复申诉 | 后端应校验（如已有未处理的同类型申诉 → 提示"你已有一个待处理的申诉，请等待教师回复"） |
| 申诉理由为空 | disabled 按钮 + 提示 |
| 提交中 | loading |
| 提交成功 | 弹窗关闭，申诉历史刷新，显示新申诉（状态：处理中） |

### 8.2 教师处理申诉

| 情况 | 行为 |
|------|------|
| 通过申诉 + 未填新分数 | 如果类型是"教师评分申诉"或"互评申诉" → 要求填写调整后的分数，否则 disabled 按钮 |
| 驳回申诉 | 回复内容必填，解释原因 |
| 处理提交中 | loading + "处理中..." |
| 处理成功后 | 学生收到通知 + 成绩自动重算（如果有调分） + 申诉从"待处理"列表消失 |
| 被其他人先处理了 | 提示"该申诉已被处理" + 列表刷新（防止两人同时处理同一条） |

# 模块 9：通知系统

### 9.1 通知触达

| 通知类型 | 触发时机 |
|---------|---------|
| 新申诉 | 学生提交申诉 → 该班教师收到 |
| 申诉已处理 | 教师处理申诉 → 该学生收到 |
| 子任务待审批 | 学生提交 → 组长收到 |
| 子任务被打回 | 组长打回 → 子任务负责人收到 |
| 子任务审批通过 | 组长通过 → 子任务负责人收到 |
| 成绩已发布 | 教师发布 → 全班收到 |
| 互评已开启 | 任务截止 → 全班收到 |
| 互评即将截止 | 截止前 1 小时 → 未完成互评的学生收到 |
| 被移出班级 | 教师移除 → 该学生收到 |
| 被移出小组 | 组长移除 → 该学生收到 |
| 入组申请通过/拒绝 | 组长操作 → 申请人收到 |
| 任务截止前提醒 | 截止前 24 小时 → 未完成的学生收到 |

### 9.2 通知展示

| 情况 | 行为 |
|------|------|
| 有未读通知 | 侧边栏/顶部出现红色徽标数字 |
| 通知列表 | 按时间倒序，未读的通知左边有蓝色标记点 |
| 点击通知 | 跳转到相关页面（如"新申诉"→ 跳到申诉 Tab） |
| 标记已读 | 点击即标记 + 蓝色标记点消失 |
| 全部已读 | 一键操作，按钮在列表顶部 |
| 通知过多 | 分页加载，每页 20 条 |
| 无通知 | 显示"暂无通知" |
| 通知加载失败 | 显示"加载失败 [重试]" |

# 模块 10：多角色切换

### 10.1 角色切换

| 情况 | 行为 |
|------|------|
| 教师端 → 学生端切换 | Shell 顶部头像区显示"切换到学生端"按钮。点击后：当前页面不变，但 token 切换为学生 token，路由重定向到学生首页 |

# 模块 11：全局 UI 细节

### 11.1 加载状态

| 区域 | 加载方式 |
|------|---------|
| 整页加载 | 骨架屏（Skeleton），不是空白页 + 转圈 |
| 表格/列表加载 | Element Plus `v-loading` 指令，表格区域覆盖半透明遮罩 + 转圈 |
| 按钮加载 | 按钮文字变为"处理中..." + loading 图标 + disabled |
| 卡片加载 | 骨架屏 |
| 弹窗加载 | 弹窗内容区转圈 |

### 11.2 空状态

| 场景 | 空状态文案 + 行动 |
|------|-----------------|
| 教师无班级 | "你还没有创建班级 [创建第一个班级]" |
| 班级无学生 | "还没有学生加入 邀请码：ABC123 [复制]" |
| 班级无任务 | "还没有发布任务 [发布第一个任务]" |
| 任务无子任务 | "组长还没有拆解子任务" |
| 无通知 | "暂无通知" |
| 搜索无结果 | "未找到匹配的结果 [清空筛选]" |
| 无申诉 | "暂无申诉记录" |
| 无互评 | "暂无互评记录" |

### 11.3 错误状态

| 场景 | 行为 |
|------|------|
| 网络断开 | 页面顶部出现黄色横幅"网络连接已断开，数据可能不是最新。 [重试]" |
| 接口 401 (token 过期) | 弹窗提示"登录已过期，请重新登录" → 跳转登录页 |
| 接口 403 (无权限) | 提示"你无权执行此操作" + 跳回首页 |
| 接口 404 | 提示"请求的资源不存在" |
| 接口 500 | 提示"服务器异常，请稍后重试。 [重试]" |
| 接口超时 | 提示"请求超时，请重试。 [重试]" |
| 接口返回未知错误 | 提示"操作失败：{错误信息}" |
| 重试按钮 | 所有错误提示都附带 [重试] 按钮 |

### 11.4 浏览器行为

| 场景 | 行为 |
|------|------|
| 用户点击浏览器"返回" | 页面正常回退，保持之前的状态（滚动位置、Tab 选择、筛选条件） |
| 用户刷新页面 (F5) | 重新加载，保持当前路由，但筛选/滚动状态丢失（可接受） |
| 用户在两个标签页打开同一系统 | 各自独立登录态（已实现） |
| 用户关闭标签页 | 不需要额外处理 |

### 11.5 移动端注意事项

| 场景 | 行为 |
|------|------|
| 侧边栏 | 小屏自动隐藏，点击汉堡菜单展开 |
| 表格 | 横向滚动（或改为卡片布局） |
| 弹窗 | 全屏弹窗（移动端） |
| 日期选择器 | 使用原生日期选择器或移动端友好的 picker |
| 长列表 | 触底加载更多（而非分页按钮） |

# 检查清单：实施时逐项验证

在实现每个页面时，必须验证以下所有状态：

```
□ 正常状态（有数据）
□ 空状态（无数据 + 引导文案 + 行动按钮）
□ 加载状态（骨架屏 / loading）
□ 错误状态（网络断开 / 500 / 超时 / 权限不足）
□ 边缘数据（只有 1 条 / 数据刚好一页 / 数据多页）
□ 操作确认（删除 / 解散 / 刷新邀请码 / 移除成员）
□ 操作反馈（成功提示 / 失败提示 + 重试）
□ 按钮状态（正常 / loading / disabled）
□ 多人并发（乐观锁冲突提示）
□ 数据过期（token 过期 / 邀请码过期）
□ 输入校验（为空 / 格式错误 / 超长）
□ 文件上传（超大 / 格式不支持 / 上传中 / 上传失败）
```
