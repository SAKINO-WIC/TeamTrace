# TeamTrace UX 遗漏分析 — 产品经理视角

> 以"一个学期完整使用"为时间轴，逐帧审视。找出之前三份文档没有覆盖的场景。

---

## 一、用户生命周期遗漏

### 1.1 教师：创建班级后 → 学生加入前（空窗期）

| 场景 | 问题 | 需要设计 |
|------|------|---------|
| 教师刚创建班级，分享了邀请码 | 教师接下来该干什么？不能干等学生加入 | 班级详情页在"0学生"状态下，应引导教师："班级已创建！现在你可以：① 预设分组 ② 提前发布第一个任务 ③ 查看邀请码状态"。**不能让教师面对一个空页面发呆。** |
| 教师提前发布了任务 | 学生还没有加入，任务列表里"进度 0%" | 任务列表在学生加入前显示"等待学生加入（当前 0 人）"而非"进度 0%"。**0% 暗示失败，"等待学生"是正常状态。** |
| 邀请码过期了但还没人加入 | 教师可能忘记了 | **主动通知**："你的班级「软件工程A班」邀请码已过期，还没有学生加入。[生成新邀请码]" |

### 1.2 学生：加入班级后 → 加入小组前（空窗期）

| 场景 | 问题 | 需要设计 |
|------|------|------|
| 学生用邀请码加入了班级 | 然后呢？没有小组，看不到任务 | 班级详情页在"未分组"状态下显示："你已成功加入班级！等待老师分配小组，或 [自己创建小组]" |
| 班级有任务但学生未入组 | 任务详情页显示什么？ | 显示"你需要先加入一个小组才能查看任务。 [去加入小组]" |
| 老师锁定了分组、学生没组 | 学生无法自己建组 | 显示"教师已锁定分组，请联系老师将你分配到小组" |

### 1.3 学期结束（之前完全没设计）

| 场景 | 问题 | 需要设计 |
|------|------|------|
| 学期结束，班级状态变为"已结束" | 学生还能看到什么？ | 班级变为**只读模式**：所有历史任务、成绩、互评可查看，但不能提交新内容。页面顶部显示横幅："本学期已结束，以下为历史记录。" |
| 教师想复用上一学期的班级 | 重新建班还是复制？ | **提供"复制班级"功能**：复制班级结构（名称、小组设置）和任务模板（不含学生数据），生成新学期班级。 |
| 教师想导出期末成绩 | 成绩怎么批量导出？ | 课程总评页提供"导出 Excel"按钮，导出格式：学号/姓名/各任务成绩/互评均分/教师评分/课程总评 |
| 教师解散班级 | 学生视角什么变化？ | 学生列表中的班级卡片显示"已解散"灰色标签，不可进入。如果学生正在浏览该班级 → 页面弹出提示"该班级已被教师解散"，自动跳回首页。 |

## 二、多角色协作冲突

### 2.1 教师操作 vs 学生操作（并发冲突）

| 场景 | 问题 | 需要设计 |
|------|------|------|
| 教师修改任务截止时间，学生正在提交 | 学生以旧截止时间为准提交了 → 后端校验可能是旧时间 | **前端在校验截止时间时使用服务器返回的最新数据**（提交前先 GET 一次确认截止时间），如果发现截止时间已变更 → 提示"任务截止时间已更新为新时间，请确认。" |
| 教师将学生从 A 组移到 B 组，学生正在 A 组任务页提交 | 学生提交时已经不属于 A 组 | 提交时后端返回"你已不在该小组" → 前端显示"你已被教师移动到另一小组，请刷新页面。" + 自动跳转到正确的小组页面 |
| 教师删除任务，学生正在任务详情页 | 学生点击提交时任务已不存在 | 后端返回 404 → 前端显示"该任务已被教师删除" → 自动跳回班级详情 |
| 教师锁定分组，组长正在审批入组申请 | 审批操作应该失败 | 组长点击[通过]时后端返回"分组已锁定" → 前端提示并隐藏审批按钮 |
| 两个学生同时认领同一子任务 | 乐观锁冲突 | 先到先得。失败者看到"该子任务已被[某同学]认领" + 列表自动刷新 + 子任务从"可认领"区消失 |

### 2.2 组长操作 vs 组员操作

| 场景 | 问题 | 需要设计 |
|------|------|------|
| 组长打回子任务，组员正在修改后重新提交 | 组员刚写好的内容被覆盖 | 组员打开子任务详情时如果子任务状态从"待审批"变回"进行中"→ **实时更新状态**（如果 WebSocket 未实现，则在组员点击提交前做一次状态检查） |
| 组长删除子任务，组员正在填写提交内容 | 组员写完发现子任务没了 | 同上——提交前检查子任务是否存在。不存在 → 提示并关闭弹窗 |
| 组长移除组员 | 被移除的组员如果正在该任务页面 | 页面顶部弹出"你已被移出小组" → 跳回班级详情 |
| 组长自己退出（如果允许） | 谁来当新组长？ | ⚠️ PRD 说组长不可自己退出。但如果教师强制移除组长 → **教师需要同时指定新组长**（界面提示："该学生是组长，移除前请先指定新组长"） |

## 三、时间维度遗漏

### 3.1 截止时间相关

| 场景 | 问题 | 需要设计 |
|------|------|------|
| 任务截止时间是 23:59，学生 23:58 提交 | 时间判断精度 | **统一使用服务器时间**（不是客户端时间，防止用户改系统时间作弊）。所有截止时间显示都应标注"以服务器时间为准"。 |
| 学生所在时区与教师不同 | 截止时间显示混乱 | 当前版本：显示截止时间时注明时区，如"5月20日 23:59 (UTC+8)" |
| 截止前 5 分钟 | 学生可能忘记 | 页面顶部倒计时条变红 + 闪烁："距离截止还有 4分32秒" |
| 截止后学生仍可提交（逾期提交） | 提交时如何提示？ | 弹窗提示："该任务已逾期。你仍可提交，但组长可能不接受逾期内容。确定提交？" [取消] [仍要提交] |
| 互评截止后教师延长时间 | 学生需要重新进入 | 已关闭的互评页面自动刷新 → 显示"互评时间已延长" + 新的倒计时 |

### 3.2 长期不活跃

| 场景 | 问题 | 需要设计 |
|------|------|------|
| 学生加入班级后 30 天未登录 | 教师看到"僵尸学生" | 学生列表可显示"最后活跃时间"，超过 14 天未活跃的标记为"不活跃" |
| 教师发布任务后 7 天无人认领 | 任务可能设置不合理 | 不需要系统干预，但任务详情页可以提示教师"该任务已发布 7 天，尚无学生认领。是否需要调整？" |

## 四、数据生命周期遗漏

### 4.1 软删除与恢复

| 场景 | 问题 | 需要设计 |
|------|------|------|
| 教师解散班级（30天可恢复） | 解散后的班级在哪显示？ | 班级列表底部增加一个折叠区"已解散的班级"，显示解散时间 + "剩余 X 天可恢复" + [恢复] 按钮 |
| 教师删除任务 | 学生还能看到已提交的内容吗？ | 删除的任务学生端不再显示。如果有必要保留，教师应"归档"而非"删除"（但当前只有删除） |
| 学生被移出班级 | 历史数据保留但不可访问 | 学生在"已结束/已离开"的班级列表中仍可看到该班级（只读），但不能操作 |

### 4.2 数据导出

| 场景 | 需要设计 |
|------|---------|
| 教师导出班级成绩单 | 课程总评页增加"导出成绩单"按钮 → 生成 Excel（学号/姓名/各任务成绩/互评/教师评分/课程总评） |
| 教师导出任务报告 | 报告中心已存在但功能不完整 → 任务详情页增加"导出报告"按钮 → PDF/Excel |
| 学生导出个人成绩 | 学生成绩页增加"导出我的成绩"按钮 |
| 管理员导出操作日志 | 日志页已支持 CSV 导出 |

## 五、非功能性 UX 遗漏

### 5.1 效率操作

| 场景 | 需要设计 |
|------|---------|
| **键盘快捷键** | 至少实现：`Enter` 提交表单、`Escape` 关闭弹窗、`Ctrl+S` 保存、`/` 聚焦搜索框 |
| **批量操作** | 教师学生管理页：勾选多个学生 → [批量移除]；评分中心：连续打分时不弹窗，直接行内编辑 + Tab 跳下一行 |
| **搜索全局化** | 教师端顶部搜索框：输入学生姓名 → 跨班级搜索 → 显示"张三 — 软件工程A班 / 数据结构B班" |
| **快速跳转** | 教师在任何页面都可以通过 Ctrl+K（或顶部搜索框）快速跳转到任意班级/任务 |

### 5.2 状态感知

| 场景 | 需要设计 |
|------|---------|
| **浏览器 Tab 标题** | 动态更新：有新通知时标题变为 `(3) TeamTrace`，在任务页时标题变为 `文献综述报告 - TeamTrace` |
| **页面标题与 URL 同步** | 当前 Tab 切换时，URL 不变（因为是同一个路由），但浏览器后退按钮应回到上一个 Tab——需要 `history.pushState` |
| **未保存提醒** | 在表单页（创建班级/发布任务/提交子任务）离开时 → `beforeunload` 事件弹窗"你有未保存的内容，确定离开？" |
| **Session 过期提醒** | Token 7天过期，过期前 1 小时弹出提醒"你的登录即将过期，建议保存当前工作。" |

### 5.3 无障碍

| 场景 | 需要设计 |
|------|---------|
| 颜色对比度 | 所有文字/背景组合满足 WCAG AA 标准（对比度 ≥ 4.5:1） |
| 键盘导航 | 所有交互元素可通过 Tab 键访问，焦点状态有明显视觉指示 |
| 屏幕阅读器 | 关键元素添加 `aria-label`，状态变化使用 `aria-live` |

### 5.4 移动端（PRD 要求移动端 H5）

当前 3 份文档全部以桌面端为基准。移动端需要针对性设计：

| 桌面端组件 | 移动端替代 |
|-----------|-----------|
| 侧边栏 | 底部 Tab Bar（首页/任务/班级/我的） |
| 大表格 | 卡片列表（每行变成一张卡片） |
| Tab 切换 | 顶部滑动 Tab 或下拉选择 |
| 日期选择器 | 原生日期控件 |
| 弹窗 | 底部 Sheet（从下往上滑出） |
| 拖拽移动成员 | 改为下拉选择目标组 |

## 六、情感设计遗漏

### 6.1 里程碑庆祝

| 时刻 | 设计 |
|------|------|
| 教师第一个班级创建成功 | 🎉 全屏轻动画 + "你的第一个班级已就绪！接下来..." |
| 第一个学生加入 | 班级详情页顶部出现一次性横幅"🎉 第一个学生加入了！" |
| 所有学生完成某任务 | 任务详情页顶部"✅ 全班已完成！" |
| 学生完成所有子任务 | 任务详情页"🎉 全部完成！你的努力被看见了。" |
| 学生获得高分 | 成绩发布时如果 ≥90 分，成绩卡片有特殊的金色边框 |

### 6.2 焦虑缓解

| 时刻 | 设计 |
|------|------|
| 多个任务快截止 | 工作台用**平静的蓝色**提示而非**警告的红色**："你有 2 个任务将在 3 天内截止。[查看]" |
| 成绩不理想 | 成绩页不只用红色表示低分。低于 60 分时显示"别灰心，每次努力都算数。如有疑问可提交申诉。" |
| 申诉被驳回 | 不只显示"已驳回"，而是显示教师的回复内容 + "如有进一步疑问，请联系老师。" |

### 6.3 空状态人格化

| 当前（冷漠） | 改为（有温度） |
|-------------|---------------|
| "暂无数据" | "这里还空空的，等待你的第一个班级。" |
| "无搜索结果" | "没有找到匹配的结果。试试换个关键词？" |
| "暂无通知" | "一切平静，没有新消息。" |
| "暂无申诉" | "没有申诉就是最好的消息 ✌️" |

## 七、功能细节查漏补缺

### 7.1 PRD 中提到但之前设计未覆盖的

| PRD | 功能 | 缺失设计 |
|-----|------|---------|
| FR-02 | 忘记密码 | 登录页"忘记密码"→ 输入手机号 → 收到验证码 → 重置密码。当前只能显示"联系管理员" |
| FR-11~13 | 学生自建分组 | 后端有接口但前端无入口。应在学生班级页面的"小组"Tab 中增加"创建小组"按钮 |
| FR-23 | WebSocket 实时推送 | 未实现。退一步方案：**每 30 秒自动轮询**关键数据（进度百分比、新通知数量），在页面顶部显示"数据已自动更新" |
| FR-34,35 | 报告导出 | 前端有 html2pdf.js，但缺少报告数据接口 |

### 7.2 代码审查中发现的遗漏

| 文件 | 问题 | 需要设计 |
|------|------|---------|
| `StudentTaskDetailView.vue` (1930行) | 一个文件承担了子任务 CRUD + 进度 + 审批 + 草稿。拆分后每个子模块需要独立的状态管理 | 拆分方案：`SubtaskList.vue` + `SubtaskSubmitModal.vue` + `SubtaskReviewPanel.vue` + `TaskProgressCard.vue` |

## 八、最终检查清单

当实现任何页面时，必须回答以下全部问题：

```
□ 首次使用（空状态）：用户看到什么引导？
□ 日常使用（有数据）：最重要的信息在第一屏吗？
□ 加载中：骨架屏还是转圈？
□ 错误：有具体原因 + 重试按钮吗？
□ 操作确认：删除/解散/移除有确认弹窗吗？
□ 操作反馈：成功/失败有提示吗？失败可重试吗？
□ 撤销：关键操作可撤销吗（5秒内）？
□ 数据过期：Token过期、邀请码过期、任务截止怎么处理？
□ 并发冲突：两人同时操作怎么处理？
□ 离开页面：有未保存内容的提醒吗？
□ 空窗期：创建后等待他人参与的过渡状态怎么展示？
□ 结束状态：学期结束/任务完成后的只读模式怎么展示？
□ 情感温度：成功有庆祝吗？失败有鼓励吗？空状态有温度吗？
□ 效率操作：有键盘快捷键吗？有批量操作吗？
□ 移动端：在小屏幕上可用吗？
```

## 总结：之前遗漏的关键项（按严重程度排序）

| # | 遗漏项 | 严重程度 | 决策 (2026-05-12) |
|---|--------|:------:|------|
| 1 | **学期结束流程** | 🔴 致命 | ✅ 采用：手动归档 + 学期后30天自动归档 |
| 2 | **并发冲突处理** | 🔴 致命 | ✅ 采用：前端防御性检查（提交前验证状态） |
| 3 | **空窗期引导** | 🟠 严重 | ✅ 采用：班级/任务空状态显示引导文案+行动按钮 |
| 4 | **截止时间时区+防作弊** | 🟠 严重 | ✅ 采用：服务器时间为准，显示UTC+8，逾期提交有提示 |
| 5 | **移动端适配** | 🟠 严重 | ❌ 暂缓 |
| 6 | **情感设计** | 🟡 中等 | ✅ 采用：温暖空状态文案 + 里程碑庆祝 + 焦虑缓解 |
| 7 | **键盘快捷键** | 🟡 中等 | ❌ 暂缓 |
| 8 | **忘记密码** | 🟡 中等 | ✅ 采用：管理员后台重置（登录页引导联系管理员） |
| 9 | **Session 过期提醒** | 🟢 轻微 | ✅ 采用：过期前1小时提醒 + 提交前检查 + 自动草稿保存 |
| 10 | **Tab 标题动态更新** | 🟢 轻微 | ✅ 采用：通知红点 + 页面标题动态变化 |
