# TeamTrace 前端统一布局规范草案 v1

> 状态：讨论确认后的第一版统一规则（已吸收 Apple 视觉母版，但不做生搬硬套）  
> 当前用途：先作为管理员端重构基线，再逐步映射教师端与学生端  
> 生效范围：`team-trace-frontend` 三端页面的壳层、布局、密度、视觉语言与交互层级  
> 说明：历史 `docs/reference/*` 仍保留，但只作为灵感来源，不再作为页面实现真值

---

## 1. 目标

本规范用于解决当前前端多份参考文档、设计草案和真实页面之间不统一的问题。

统一目标：

1. 三端共享一套布局语法，而不是每个页面各写各的
2. 管理员端、教师端、学生端保留角色差异，但不再割裂成三种完全不同的产品语言
3. 优先统一壳层、卡片、表格、筛选区、页头、弹窗这些高频结构
4. 管理员端先落地，教师端先保护，学生端后续按移动优先规则映射
5. 以 `docs/reference/DESIGN-apple.md` 作为视觉母版，再融合 TeamTrace 自身的玻璃液态、悬浮反馈与强选中态

---

## 2. 总体原则

### 2.1 统一方向

采用“统一壳层语法，角色化内容密度”的方案。

意思是：

1. 三端共用同一套页面骨架
2. 三端共用同一套组件层级
3. 三端根据角色调节信息密度、首屏重点和操作强度
4. 视觉母版以 Apple 风格为起点，但交互反馈按 TeamTrace 的后台工作台需求增强

### 2.2 视觉母版来源

当前确认：

`DESIGN-apple.md` 是 TeamTrace 前端后续视觉语言的主参考来源。

但必须明确：

1. 它是视觉母版，不是逐条照抄的唯一实现规范
2. TeamTrace 不是产品官网，而是多角色业务系统
3. 所以后续实现遵循的是：

`Apple 的秩序感 + TeamTrace 的工作台结构 + 受控玻璃液态 + 明显交互反馈`

### 2.3 当前优先级

1. 先定义规则，不急着继续改代码
2. 第一落地点：管理员端
3. 教师端暂时视为“保护区”，不因统一规则立即大改
4. 学生端后续按移动优先规则映射，不反向约束管理员端

### 2.4 历史文档使用规则

以下文档仅作参考，不再直接等价于实现标准：

1. `docs/reference/DESIGN-apple.md`
2. `docs/reference/frontend_demo_admin.md`
3. `docs/reference/frontend_demo_teacher*.md`
4. `docs/reference/frontend_demo_student.md`
5. 其他 reference/demo/mock 性质文档

其中：

- `DESIGN-apple.md`：作为**视觉母版**
- 其他 demo/reference：作为**页面灵感参考**

当前统一布局真值以本文件为准。

## 3. Apple 母版的吸收方式

### 3.1 直接继承的部分

以下内容直接继承自 Apple 风格母版：

1. 黑、白、浅灰的主基调
2. 单一蓝色作为核心交互色
3. 标题与正文的强秩序化字阶
4. 紧凑而克制的页头
5. 高级感来自结构、留白和层级，而不是花哨装饰
6. 左侧导航与顶部栏使用轻玻璃/半透明壳层

### 3.2 必须 TeamTrace 化改造的部分

以下内容不能照搬 Apple 原教义，而必须结合 TeamTrace：

1. **悬浮效果**
   Apple 更静态；TeamTrace 需要更明确的工作台反馈
2. **玻璃液态效果**
   Apple 更克制；TeamTrace 允许在壳层、工具栏、小卡片上更明显一些
3. **选中效果**
   Apple 选中态常较隐；TeamTrace 后台场景必须更明确
4. **表格与筛选区**
   Apple 原文更偏展示型；TeamTrace 管理员端必须信息密度更高
5. **卡片结构**
   Apple 更偏展示卡片；TeamTrace 需要工作台型卡片

### 3.3 一句话融合公式

后续统一风格不是“Apple 原样复刻”，而是：

`Apple 的视觉秩序 + TeamTrace 的后台工作流 + 玻璃液态壳层 + 强选中态`

## 4. 统一壳层规则

### 4.1 标准页面骨架

三端统一采用：

`左侧菜单栏 + 顶部页头 + 主工作区`

标准结构：

1. 左侧菜单栏：负责导航、角色标识、关键状态入口
2. 顶部页头：负责页面标题、上下文说明、一级操作
3. 主工作区：负责筛选、摘要、主数据区、明细区

### 4.2 左侧菜单栏

统一要求：

1. 固定在左侧
2. 支持折叠
3. 展开宽度建议：`240px - 256px`
4. 折叠宽度建议：`72px - 80px`
5. 一级菜单必须清晰可点击，不做过多装饰
6. 当前激活项必须有高对比状态，不靠仅改变字体粗细表达
7. 菜单栏可使用更明显的半透明玻璃液态效果，但必须保证文字与图标对比度

### 4.3 顶部页头

统一要求：

1. 高度紧凑，优先做工作区页头，不做大海报
2. 左侧：页面标题 + 一句话描述
3. 右侧：高频操作按钮、刷新、退出、切换等
4. 页头不承担大量统计职责

### 4.4 主工作区

统一使用四段式：

1. 筛选区
2. 摘要区
3. 主数据区
4. 明细/辅助区

不是所有页面都必须四段都满配，但结构顺序统一。

## 5. 视觉语言统一规则

### 5.1 总体视觉方向

统一基调：

`克制、现代、秩序化、轻玻璃感`

更准确地说：

`Apple 式克制秩序 + TeamTrace 式工作台效率`

不是营销官网风，也不是厚重企业 OA 风，也不是纯展示型 Apple 官网照搬。

### 5.2 玻璃液态效果使用规则

你已经明确希望保留“玻璃液态效果”，但必须受控使用。

允许使用较明显玻璃液态效果的位置：

1. 左侧菜单栏外层
2. 顶部页头
3. 筛选工具栏
4. 小型摘要卡片
5. 弹窗或抽屉头部
6. 菜单激活区的底板

不建议大面积使用玻璃效果的位置：

1. 大表格主体
2. 长文本详情区
3. 表单主内容区
4. 高密度列表区
5. 学生端长内容阅读区

原因：

1. 玻璃效果适合“壳层”和“轻工具层”
2. 数据阅读区更适合实体底板，避免对比度下降

液态感控制要求：

1. 允许 `backdrop-filter: blur(...) saturate(...)`
2. 允许半透明白 / 半透明深色底
3. 允许轻微高光边界
4. 不允许做成过分像果冻或彩色水滴的拟物效果
5. 不允许影响表格和正文可读性

### 5.3 悬浮效果规则

统一保留悬浮反馈，但不做夸张漂浮感。

Hover 规则：

1. 卡片：轻微上浮 `translateY(-1px ~ -2px)`
2. 阴影：略增强，不出现厚重大阴影
3. 边框：允许轻微高亮
4. 按钮：优先明度变化 + 轻微上浮
5. 表格行：优先背景弱变化，不做卡片式弹起

允许比 Apple 原风格更明显，但必须控制在“精致反馈”范围内，不做游戏化弹跳。

禁止：

1. 大幅位移
2. 强烈缩放
3. 过度渐变闪烁
4. 高噪声动效

### 5.4 选中效果规则

选中态必须明显，这是 TeamTrace 相对 Apple 原风格的强化项。

统一要求：

1. 侧边栏当前菜单项必须有明显底板
2. 激活项必须同时具备：
   - 背景变化
   - 文字颜色变化
   - 图标颜色变化
   - 必要时加入左侧高亮线或内发光边界
3. 筛选标签、分段器、Tab、列表当前项不能只靠字体加粗表达
4. 选中态优先使用品牌蓝及其透明层，不新增多彩高亮体系

推荐选中态构成：

1. 背景：蓝色透明底
2. 文字：高对比亮色或深色
3. 边界：轻边框 / 内阴影 / 左侧强调线

### 5.5 卡片式布局规则

统一采用卡片式布局，但要区分层级。

卡片分三级：

1. 壳层卡：页头、侧栏、工具栏
2. 数据卡：统计、摘要、模块入口
3. 内容卡：表格容器、详情容器、表单容器

统一要求：

1. 圆角统一，不同页面不随意漂移
2. 卡片内边距按密度等级统一
3. 卡片背景和边框层级明确
4. 内容卡优先稳定，不追求浮夸装饰

## 6. 三端密度分级

### 6.1 管理员端

定位：

`高密度后台工作台`

规则：

1. 信息密度最高
2. 说明文案最少
3. 筛选、表格、操作区优先
4. Hero 弱化，首页更像运营总览而不是品牌页
5. 玻璃效果只保留在壳层和工具栏
6. 选中态和工作流反馈必须最明确

### 6.2 教师端

定位：

`中高密度业务工作台`

规则：

1. 保留一定任务流与上下文引导
2. 比管理员端更强调处理流程
3. 允许少量场景型摘要卡
4. 表格与卡片并用，但以工作效率优先

### 6.3 学生端

定位：

`中密度任务流界面`

规则：

1. 移动优先
2. 强调任务、待办、进度、通知
3. 允许更多状态提示和引导文案
4. 不把大面积玻璃效果铺到数据区

## 7. 标准页面模板

### 7.1 总览页模板

适用于：

1. 管理员总览
2. 教师首页
3. 学生首页

推荐结构：

1. 页头
2. 关键摘要卡
3. 快捷入口区
4. 最近数据或待办区
5. 辅助说明区

控制要求：

1. 不做超大 Hero
2. 首屏必须能看到真实工作入口
3. 摘要卡不超过两排

### 7.2 工作台页模板

适用于：

1. 邀请码管理
2. 账户管理
3. 监控页
4. 日志页
5. 教师任务中心

推荐结构：

1. 页头
2. 筛选工具栏
3. 小摘要区
4. 主数据区
5. 分页 / 底部操作区

控制要求：

1. 表格或列表必须成为主视觉中心
2. 工具栏比说明区更重要
3. 尽量避免一个页面里出现过多并列大卡片

### 7.3 详情页模板

适用于：

1. 班级详情
2. 任务详情
3. 账户详情
4. 日志详情

推荐结构：

1. 顶部上下文区
2. 关键信息卡
3. 主体内容区
4. 操作区 / 侧信息区

控制要求：

1. 详情页强调层次，不强调数量
2. 主信息优先固定，次信息可折叠或下沉

---

## 8. 管理员端专项规则

管理员端是统一规则的第一落地点。

### 8.1 管理员端壳层要求

1. 左侧菜单栏保留
2. 顶部页头缩短
3. 大段品牌介绍删除或下沉
4. 侧边栏说明卡减少，只保留必要范围信息
5. 管理员端优先体现 Apple 式秩序感，但菜单激活、筛选操作、悬浮反馈必须比 Apple 原版更明显

### 8.2 管理员端首页要求

首页改为：

1. 一排紧凑 KPI
2. 一排快捷入口
3. 最近邀请码 / 账户摘要
4. 简短范围说明

不再强调：

1. 大 Hero
2. 大标题叙事
3. 大面积解释性文案

### 8.3 邀请码页要求

邀请码页作为管理员端第一优先工作台，结构固定为：

1. 筛选区
2. 小摘要卡
3. 生成邀请码工作区
4. 邀请码列表

原则：

1. 重点是“筛选 + 生成 + 列表 + 操作”
2. 生成区比说明区重要
3. 列表信息密度优先

### 8.4 账户页要求

结构固定为：

1. 筛选区
2. 当前页摘要
3. 表格区
4. 分页区

原则：

1. 表格是主角
2. 操作按钮必须紧凑、稳定、可确认
3. 不用堆过多装饰性模块

### 8.5 监控与日志页要求

统一按“读数据的工作台”做：

1. 先筛选
2. 再看数据
3. 再看详情

日志页尤其强调：

1. 高密度表格
2. 详情抽屉
3. before / after 区块化阅读

## 9. 教师端保护线

教师端昨天刚做过大调整，因此本轮只定义规则，不直接回改。

保护线如下：

1. 不立刻重写教师端整体壳层
2. 不因为管理员端压缩密度，就直接套用到教师端
3. 教师端现有任务详情、工作台逻辑优先保留
4. 后续如果要映射统一规则，只做“局部对齐”，不做推倒重来

本轮教师端只接受三类调整：

1. 尺寸 token 对齐
2. 交互反馈对齐
3. 壳层语言轻微对齐

本轮教师端暂不接受：

1. 信息架构重做
2. 页面整体换骨架
3. 把已稳定工作流改成管理员风格

## 10. 统一颜色与状态基线

### 10.1 基础颜色

全局主基调以 Apple 文档为母版：

1. 黑：`#000000`
2. 浅灰：`#f5f5f7`
3. 近黑文字：`#1d1d1f`
4. 白：`#ffffff`
5. 主交互蓝：`#0071e3`

### 10.2 TeamTrace 化补充

为了适配后台工作台与玻璃液态效果，允许增加：

1. 蓝色透明层：用于选中态底板
2. 半透明浅色壳层：用于玻璃工具栏和侧栏
3. 半透明深色壳层：用于暗色模式下壳层
4. 少量危险色、成功色、警示色：仅用于业务状态

规则：

1. 主交互色仍以蓝色为唯一主色
2. 状态色只用于状态，不用于品牌表达
3. 禁止在管理员端引入大面积多彩装饰

## 11. 统一尺寸与间距基线

### 11.1 间距等级

建议统一：

1. `8`：超小间距
2. `12`：小间距
3. `16`：标准控件间距
4. `20`：区块内边距
5. `24`：卡片标准内边距
6. `32`：大区块间距

### 11.2 圆角等级

建议统一：

1. 控件圆角：`10px - 12px`
2. 卡片圆角：`16px - 18px`
3. 大容器圆角：`20px - 24px`
4. 胶囊按钮：`999px`

### 11.3 标题层级

建议统一：

1. 页头主标题：`24px - 28px`
2. 模块标题：`18px - 20px`
3. 数据值大号：`24px - 30px`
4. 说明文字：`13px - 14px`
5. 标签/辅助信息：`12px`

管理员端禁止再出现过大的 `32px+` 主标题作为默认状态。

## 12. 统一组件规则

后续组件抽离建议围绕这几类：

1. Shell：角色壳层
2. Toolbar：筛选与工具栏
3. MetricCard：摘要卡
4. DataCard：数据承载卡
5. TableSection：表格区容器
6. DetailDrawer：详情抽屉
7. StatusBadge：状态徽标
8. ActionBar：行内操作区

统一原则：

1. 先统一结构，再统一样式
2. 组件名称尽量表达职责，不表达某一页
3. 相同交互不要在三端重复发明

本规范确认后，落地顺序固定如下：

1. 管理员端壳层压缩与结构统一
2. 管理员总览页重构
3. 管理员邀请码页重构

## 14. 当前冻结结论

本轮先确认以下设计决策：

1. 保留左侧菜单栏
2. 保留卡片式布局
3. `DESIGN-apple.md` 作为视觉母版
4. 保留轻玻璃液态效果，并允许在壳层上比 Apple 原版更明显
5. 玻璃效果只用于壳层和轻工具层，不铺满数据区
6. 悬浮效果保留，但保持克制
7. 选中效果必须明显
8. 管理员端作为第一落地点，教师端暂缓大改

---

## 15. 一句话总结

TeamTrace 前端后续统一为：

`Apple 视觉母版 + 左侧菜单 + 紧凑页头 + 卡片工作台 + 受控玻璃液态 + 明显选中态 + 角色化密度`

其中：

1. 管理员端最克制、最紧凑、最像后台
2. 教师端最强调业务处理流
3. 学生端最强调任务执行流

这就是后续所有页面优化的统一基线。
