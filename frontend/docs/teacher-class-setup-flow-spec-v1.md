# 教师端班级建成链交互规格 v1

> 适用范围：教师端从首次进入工作台到一个班级具备继续管理条件的主链路。
> 本规格只使用当前真实接口能力，不新增后端接口，不改数据库，不迁移 Pure/Vben 模板工程。

## 1. 目标与边界

### 1.1 目标

把教师端的“创建班级”改造成下一步明确的任务链，而不是让教师在多个统计卡片、列表页、详情页之间自行判断。

班级建成链定义为：

```text
教师工作台空状态
  -> 创建第一个班级
  -> 生成或展示班级邀请码
  -> 等待学生加入
  -> 班级详情成为主工作区
  -> 学生管理 / 分组管理 / 任务管理入口一致可达
```

### 1.2 不做的事

- 不迁移到 Pure Admin 或 Vben Admin 工程。
- 不新增接口，不修改接口字段，不改数据库结构。
- 不处理评分中心、申诉中心、报告中心、任务详情深层链路。
- 不在班级列表页展示真实接口没有返回的假字段，例如邀请码、任务数、小组数。
- 不通过大面积统计卡片堆叠解决引导问题。

### 1.3 可复用接口

| 能力 | 当前接口 | 前端服务 |
| --- | --- | --- |
| 查询教师班级列表 | `GET /api/teacher/classes` | `fetchTeacherClasses` |
| 查询已归档班级 | `GET /api/teacher/classes?status=archived` | `fetchTeacherArchivedClasses` |
| 创建班级 | `POST /api/teacher/classes` | `createTeacherClass` |
| 查询班级详情 | `GET /api/teacher/classes/{classId}` | `fetchTeacherClassDetail` |
| 生成或刷新班级邀请码 | `POST /api/teacher/classes/{classId}/invite-codes` | `generateTeacherClassInviteCode` |
| 查询学生分页 | `GET /api/teacher/classes/{classId}/students` | `fetchTeacherClassStudents` |
| 查询学期小组 | `GET /api/teacher/classes/{classId}/groups` | `fetchTeacherClassGroups` |
| 创建/移动学期小组 | `POST/PUT /api/teacher/classes/{classId}/groups...` | `createTeacherClassGroup`, `moveTeacherClassGroupMember`, `addStudentToGroup` |
| 查询班级任务 | `GET /api/teacher/classes/{classId}/tasks` | `fetchTeacherClassTasks` |
| 发布班级任务 | `POST /api/teacher/classes/{classId}/tasks` | `createTeacherClassTask` |

### 1.4 当前代码审计结论

- `TeacherHomeView.vue` 已有首次空状态，但主按钮只进入 `/teacher/classes`，不能直接带出创建流程。
- `TeacherClassesView.vue` 创建成功后只把班级插入列表，没有立即生成邀请码，也没有把“下一步：发邀请码”变成主反馈。
- `TeacherClassDetailView.vue` 已能刷新/复制邀请码，但对 `studentCount = 0`、无有效邀请码、邀请码过期、等待学生加入没有任务链式引导。
- `TeacherStudentsView.vue`、`TeacherGroupsView.vue`、`TeacherTasksView.vue` 都有 `TeacherClassNav`，但子页未统一传入计数，返回按钮也普遍回到班级中心，而不是回到当前班级详情。
- `TeacherGroupsView.vue` 空状态里绑定了不存在的 `openCreateGroupDialog`，实际函数是 `openCreateDialog`；后续实现时应一并修正。
- `TeacherClassNav.vue` 作为班级内一致导航可保留，但应从“卡片式跳转入口”压缩为稳定的班级二级导航，避免详情页再重复一组快捷按钮。

## 2. 设计原则

1. 每个屏幕只回答一个教师问题：我现在该做什么。
2. 主按钮只保留一个，必须指向当前链路的下一步。
3. 次级操作可以存在，但不能抢过主按钮。
4. 空状态不是说明文案，应包含当前状态、下一步动作、不能做什么。
5. 加载、错误、成功反馈必须落在当前页面内，不能只依赖短暂 toast。
6. 班级详情是建成链主工作区，学生、分组、任务是它的子工作区。
7. 子页返回统一回到当前班级详情，不回到班级中心。
8. 邀请码是一次性、24 小时有效的当前真实规则，页面必须把“需要刷新新码”作为正常动作。

## 3. 状态机

| 状态 | 判定 | 主动作 | 下一屏 |
| --- | --- | --- | --- |
| 没有班级 | `classCount = 0` | 创建第一个班级 | 班级中心创建面板 |
| 已创建班级但无有效邀请码 | `studentCount = 0` 且 `activeInviteCode` 为空 | 生成邀请码 | 当前班级详情 |
| 有有效邀请码但 0 学生 | `activeInviteCode` 有值，`studentCount = 0` | 复制邀请码 | 当前班级详情 |
| 有学生但无小组 | `studentCount > 0` 且 `groupCount = 0` | 创建第一个小组 | 分组管理 |
| 有学生有小组但无任务 | `studentCount > 0` 且 `groupCount > 0` 且 `taskCount = 0` | 发布第一个任务 | 任务管理 |
| 班级可用 | `studentCount > 0` 且至少完成小组或任务准备 | 进入班级工作区 | 班级详情 |

说明：当前 `GET /teacher/classes/{classId}` 只返回有效未过期的邀请码。若接口返回 `activeInviteCode = null`，前端只能判定“当前没有有效邀请码”，不能区分未生成、已使用、已过期或已刷新失效。页面文案应写成“当前没有有效邀请码，请生成新邀请码”。

## 4. 屏幕规格

### 4.1 教师工作台：首次空状态

**用户目标**

教师首次进入后知道必须先创建班级，而不是看到统计面板。

**主按钮**

- `创建第一个班级`
- 跳转：`/teacher/classes?create=1`
- 到达后班级中心直接展开创建面板。

**次级操作**

- 暂不展示评分、申诉、报告入口。
- 可保留“刷新”但弱化，不进入主视觉区。

**空状态**

- 标题：`先创建一个班级`
- 正文只保留三步：创建班级、生成邀请码、学生加入后再分组/发布任务。
- 不展示四张 KPI 卡片。

**加载状态**

- 首次加载显示 2 到 3 条骨架，不显示空文案。
- 加载失败时不伪装成没有班级。

**错误状态**

- 文案：`无法加载教师工作台，请重新加载或稍后再试。`
- 主按钮：`重新加载`
- 次级操作：`进入班级中心`

**成功状态**

- 如果已有班级，工作台第一块显示“待建成班级”队列，优先展示 `0 学生`、`无有效邀请码`、`无任务` 的班级。
- 如果没有待建成班级，再展示申诉、截止、活跃班级等日常待办。

**跳转规则**

- 点击待建成班级，进入 `/teacher/classes/{classId}`。
- 点击日常待办，进入对应工作区。

**验收标准**

- 空账号教师首次进入 `/teacher`，首屏唯一主动作是创建第一个班级。
- 接口失败时不会显示“你还没有班级”。
- 已有班级但 0 学生时，工作台能提示“等待学生加入”或“生成邀请码”。

### 4.2 班级中心：创建入口与成功反馈

**用户目标**

教师能快速创建班级，并在创建后立即得到可分享的邀请码或明确进入详情生成邀请码。

**主按钮**

- 无班级或从 `?create=1` 进入时：创建面板展开，主按钮为 `创建班级并生成邀请码`。
- 已有班级时：页头主按钮仍为 `创建班级`。

**次级操作**

- 搜索、学期筛选、归档筛选保留，但不能位于创建面板之前抢占首次视线。
- 已归档班级恢复保留。

**表单字段**

- 班级名称：必填，最长 100。
- 学期：必填，必须控制在后端 20 字符限制内。
- 小组人数：不再使用 `2-5` 固定范围；第一轮不新增人数设置入口，默认按“至少 1 人即可成组”处理，一人成组时互评不适用。

**成功状态**

推荐实现：

1. 创建班级成功后，立即调用当前已有的 `generateTeacherClassInviteCode(classId)`。
2. 在当前页展示成功面板：
   - 班级名称、班级编号、有效期。
   - 邀请码大号显示。
   - 主按钮：`复制邀请码`
   - 次级操作：`进入班级详情`、`稍后再说`
3. 点击 `进入班级详情` 跳转 `/teacher/classes/{classId}`，详情页顶部保留“刚创建”成功提示。

备选实现：

- 如果后续不希望创建班级自动生成邀请码，则创建成功后跳转到班级详情，并把主按钮固定为 `生成邀请码`。此方案需要用户确认，因为会少一步“创建成功后直接看到邀请码”。

**加载状态**

- 创建中禁用表单和按钮。
- 列表刷新时保留已有列表，局部显示刷新中。

**错误状态**

- 班级创建失败：错误显示在表单顶部，保留已填写内容。
- 邀请码生成失败：班级已经创建成功，成功面板改为“班级已创建，但邀请码生成失败”，主按钮为 `重新生成邀请码`，次级操作 `进入班级详情`。

**跳转规则**

- 整张班级卡片进入 `/teacher/classes/{classId}`。
- 已归档班级点击恢复，不进入详情。
- 从工作台 `?create=1` 进入后，关闭创建面板仍停留班级中心。

**验收标准**

- 从工作台点击 `创建第一个班级` 后，不需要再找一次创建按钮。
- 创建成功后能看到邀请码或看到明确的生成邀请码主动作。
- 邀请码生成失败不会让教师误以为班级创建失败。

### 4.3 班级详情：邀请码与 0 学生引导

**用户目标**

教师进入一个新班级后能完成“把学生带进来”的动作，并知道学生加入前哪些操作可以先做。

**主区域顺序**

1. 班级标题与返回。
2. 班级建成状态条。
3. 邀请码面板。
4. 下一步面板。
5. 班级内导航：概览 / 学生 / 分组 / 任务。

**主按钮规则**

| 条件 | 主按钮 |
| --- | --- |
| 无有效邀请码 | `生成邀请码` |
| 有有效邀请码且 0 学生 | `复制邀请码` |
| 有学生但无小组 | `创建第一个小组` |
| 有学生有小组但无任务 | `发布第一个任务` |
| 已有任务 | `进入任务管理` |

**次级操作**

- `刷新邀请码`：只有在有邀请码或生成失败后展示，且说明旧码会失效。
- `查看学生名单`：始终可见，但 0 学生时是次级。
- `管理分组`、`管理任务`：可见，但在 0 学生时标注“可先预设，学生加入后生效”。
- `解散班级`：保留在风险区，不放在首屏主操作旁边。

**0 学生状态**

- 文案：`还没有学生加入。请先把班级邀请码发给学生。`
- 展示当前有效邀请码、有效期、复制动作。
- 允许教师提前做两件事：
  - 预设分组规则或创建空小组：如果没有学生，创建小组按钮应说明“需要先有学生才能创建含成员小组”。
  - 发布第一个任务：任务可以先发布，但列表应显示“等待学生加入”，不要用“进度 0%”表达失败。

**已有学生状态**

- 建成状态条展示：`已加入 X 人 / Y 个小组 / Z 个任务`。
- 如果学生都未分组，主动作转为 `创建第一个小组`。
- 如果已分组但无任务，主动作转为 `发布第一个任务`。

**无有效邀请码状态**

- 文案：`当前没有有效的邀请码，可能尚未生成、已过期、已使用或已被刷新。`
- 主按钮：`生成新邀请码`
- 生成成功后切换为复制状态。

**邀请码过期状态**

- 如果页面已有 `inviteExpireAt` 且时间小于当前时间，状态条显示 `邀请码已过期`。
- 主按钮：`生成新邀请码`
- 不展示过期码作为可复制内容。
- 若重新加载后接口不再返回过期码，则按“无有效邀请码”处理。

**加载状态**

- 班级详情、任务、分组并行加载。
- 邀请码面板显示骨架，不显示 `-` 作为有效邀请码。

**错误状态**

- 班级详情接口失败：整页错误，主按钮 `重新加载`，次级 `返回班级中心`。
- 任务或分组接口失败：班级基本信息与邀请码仍可用，失败模块内显示 `无法加载任务/分组`，可局部重试。

**成功状态**

- 刷新邀请码成功：显示新邀请码，并提示 `旧邀请码已失效`。
- 复制成功：按钮短暂变为 `已复制`，不弹阻断式对话框。
- 分组锁定切换成功：状态条内更新为 `分组已锁定` 或 `分组开放中`。

**跳转规则**

- `学生管理` -> `/teacher/classes/{classId}/students`
- `分组管理` -> `/teacher/classes/{classId}/groups`
- `任务管理` -> `/teacher/classes/{classId}/tasks`
- 所有子页返回 -> `/teacher/classes/{classId}`

**验收标准**

- 创建班级后进入详情，首屏能看到邀请码状态与下一步。
- 0 学生状态不会出现空表格作为主内容。
- 无有效邀请码时，教师能一眼找到生成新邀请码。
- 任务、分组局部失败不阻断复制邀请码。

### 4.4 学生管理：等待加入与名单管理

**用户目标**

教师确认学生是否加入，并能处理已加入学生。

**主按钮**

- 0 学生：`返回班级详情复制邀请码`
- 有学生：`刷新名单`

**次级操作**

- 0 学生时保留 `返回班级详情`，不显示筛选面板作为主内容。
- 有学生时显示搜索、状态筛选、小组筛选、分页。
- 移出学生保留二次确认。

**空状态**

- 0 学生：显示“等待学生使用邀请码加入”，主按钮回详情。
- 筛选无结果：显示“当前筛选下没有学生”，主按钮 `重置筛选`。

**加载状态**

- 表格骨架，分页保持当前页。

**错误状态**

- 学生接口失败：显示局部错误，主按钮 `重新加载名单`，次级 `返回班级详情`。
- 分组接口失败：学生名单仍显示，小组列显示 `小组信息暂不可用`。

**成功状态**

- 移出学生成功后刷新当前页。
- 如果当前页移空且仍有上一页，回退上一页再加载。

**验收标准**

- 0 学生进入学生管理页时，不出现“当前条件下还没有可显示的学生”这种筛选空态。
- 子页返回按钮回当前班级详情。

### 4.5 分组管理：有学生后建组

**用户目标**

教师把学生组织成学期固定小组，为后续任务进度和互评做准备。

**主按钮**

- 0 学生：`返回班级详情复制邀请码`
- 有学生无小组：`创建第一个小组`
- 有小组：`创建教师分组`

**次级操作**

- `锁定/解锁分组`
- `移动到其他组`
- `查看任务进度`
- `查看任务详情`

**空状态**

- 没有学生：不展示创建小组表单，说明必须先有学生。
- 有学生但无小组：列出未分组学生数量，主按钮创建第一个小组。
- 有小组但未选择任务：进度区显示“选择任务后查看小组进度”，不显示 0%。

**加载状态**

- 班级详情、学生、分组、任务并行加载。
- 进度加载失败不清空小组列表。

**错误状态**

- 创建小组失败：保留弹窗内容，并显示后端错误。
- 移动学生失败：不改变本地列表，提示重新加载。
- 分组锁定失败：按钮恢复原状态。

**成功状态**

- 创建小组后关闭弹窗，刷新学生与分组。
- 移动学生后刷新学生与分组。
- 分组锁定后同步班级详情状态。

**跳转规则**

- 返回 -> `/teacher/classes/{classId}`
- 查看任务进度 -> `/teacher/classes/{classId}/tasks/{taskId}/progress`
- 查看任务详情 -> `/teacher/classes/{classId}/tasks/{taskId}`

**验收标准**

- 0 学生不能误导教师创建空小组。
- `TeacherGroupsView.vue` 的空状态按钮能真正打开创建分组入口。
- 子页导航计数不再默认显示 0。

### 4.6 班级任务：从等待学生到发布任务

**用户目标**

教师在班级已经可用后发布和跟进任务；如果学生尚未加入，也能提前发布但不会误读为失败。

**主按钮**

- 无任务：`发布第一个任务`
- 有任务：`发布任务`

**发布表单字段**

- 任务名称、任务描述、截止时间、是否开启互评、互评时长、互评权重、教师评分权重。
- 当前真实接口不支持开始时间与任务附件，页面不得展示无法提交的开始时间或附件上传控件。

**次级操作**

- `任务详情`
- `进入评分`
- `查看申诉`
- 筛选任务、互评状态。

**空状态**

- 0 学生无任务：提示 `可以先发布任务，但学生加入后才会开始协作`。
- 0 学生有任务：任务卡片标注 `等待学生加入`，不展示“0% 进度”类失败口吻。
- 有学生无任务：提示 `发布第一个任务后，小组即可开始拆解子任务`。

**加载状态**

- 列表骨架，发布按钮禁用。

**错误状态**

- 任务列表失败：保留发布入口，说明“任务列表暂不可用”。
- 发布任务失败：保留表单内容，显示字段或接口错误。

**成功状态**

- 发布成功后关闭表单，刷新任务列表。
- 新任务在列表顶部高亮 3 秒。
- 如果班级 0 学生，成功提示中追加 `学生加入后即可开始协作`。

**跳转规则**

- 返回 -> `/teacher/classes/{classId}`
- 任务详情 -> `buildTeacherTaskDetailLocation(classId, taskId, {}, 'class-tasks')`
- 评分和申诉继续走现有跨班级中心，并带 `classId/taskId/from` 查询参数。

**验收标准**

- 没有学生时，任务页不会把“等待学生”显示成异常进度。
- 发布成功后教师能明确回到当前班级上下文。

## 5. 班级内导航规格

`TeacherClassNav` 保留为班级内唯一二级导航。

### 5.1 标签

| 标签 | 路由 | 计数 |
| --- | --- | --- |
| 班级概览 | `/teacher/classes/{classId}` | 不显示 |
| 学生管理 | `/teacher/classes/{classId}/students` | 学生总数 |
| 分组管理 | `/teacher/classes/{classId}/groups` | 小组数 |
| 任务管理 | `/teacher/classes/{classId}/tasks` | 任务数 |

### 5.2 计数策略

- 班级详情页已经并行读取详情、分组、任务，可直接传计数。
- 学生页至少传学生总数；分组/任务计数如果未加载，不显示徽标，不显示默认 0。
- 分组页用已加载的 `students/groups/tasks` 传计数。
- 任务页至少传任务数；学生/分组计数如果要显示，应使用现有详情/分组接口，不新增接口。

### 5.3 返回规则

- 所有班级子页页头返回统一为 `返回班级概览`。
- 班级中心只作为列表入口，不作为班级子页的默认返回目标。

## 7. 验收清单

### 7.1 空账号教师

- 登录后进入 `/teacher`，首屏看到 `创建第一个班级`。
- 点击后进入班级中心并自动展开创建面板。
- 创建表单失败时保留输入。

### 7.2 创建成功

- 成功后能看到班级编号。
- 能看到有效邀请码、有效期、复制按钮、刷新入口。
- 邀请码生成失败时能重试，不误报班级创建失败。

### 7.3 0 学生

- 班级详情主提示是 `等待学生加入`。
- 学生管理页显示入班引导，不显示筛选空表格。
- 分组管理页不允许误建无成员小组。
- 任务管理页允许先发任务，但文案说明学生加入后协作。

### 7.4 已有学生

- 班级详情主动作能根据分组和任务状态切换。
- 学生页能筛选、分页、查看详情、移出学生。
- 分组页能创建小组和移动成员。
- 任务页能发布和进入任务详情。

### 7.5 邀请码失效

- 页面已有过期时间时能显示已过期。
- 重新加载后没有有效码时显示 `当前没有有效邀请码`。
- 点击生成新邀请码后，旧码失效提示明确。

### 7.6 接口失败

- 工作台失败不伪装成空账号。
- 班级详情失败显示整页重试。
- 任务/分组局部失败不阻断邀请码复制。

### 7.7 路由层级

- 从班级详情进入学生、分组、任务页后，返回都回到 `/teacher/classes/{classId}`。
- 班级内导航当前态准确。
- 未知计数不显示，不显示错误的 0。

## 8. 与模板工程的关系

Pure Admin 与 Vben Admin 只作为以下层面的参考：

- 左侧导航、页头、表格密度、表单布局。
- 列表筛选区、成功反馈面板、按钮层级。
- 空状态与加载骨架的布局节奏。

它们不决定业务流，不引入新路由架构，不创建新前端工程，不替换当前 Vue 3 + Vite + Element Plus 技术栈。
