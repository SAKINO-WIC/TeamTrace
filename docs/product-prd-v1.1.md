## 1. 产品概述

### 1.1 项目背景
在高校教育中，小组作业是常见的教学形式。然而，实际协作中存在普遍痛点：分工不明确、进度拖沓、部分成员“摸鱼”导致贡献与收获不对等、教师难以量化个人贡献等。现有工具（微信群、腾讯文档、Trello等）均无法系统解决上述问题。

**TeamTrace（摸鱼终结者）** 应运而生，专注于**小组作业全流程管理**，通过任务拆解、进度跟踪、匿名互评、教师评分等功能，实现过程留痕、贡献可量化、责任可追溯。

### 1.2 产品目标
- **核心目标**：让小组作业中的分工、进度、贡献度透明化，解决“干多干少一个样”的问题。
- **用户价值**：
  - 学生：公平记录贡献，避免摸鱼；任务清晰，减少沟通成本。
  - 教师：过程可视，评分有据；减少投诉处理时间。
  - 管理员：统一管理账户，监控系统运行。

### 1.3 产品范围
**包含**：
- 学生、教师、管理员三端功能
- 班级管理、分组管理、任务管理、互评、评分、申诉、通知、报告
- 实时进度看板（WebSocket）
- 数据软删除及定时清理

**不包含**：
- 实时聊天/IM功能（沟通仍用微信）
- 在线文档协作（由腾讯文档等替代）
- AI任务拆解（预留扩展）

## 3. 功能需求

### 3.1 注册与登录

#### FR-01 用户注册
- **描述**：教师通过手机号+密码+教师邀请码注册；学生通过手机号+密码注册。
- **教师邀请码**：
  - 由管理员在后端生成，有效期30天，使用后标记已使用。
  - 管理员可撤销未使用的邀请码。
- **输入**：手机号（11位）、密码（8-20位，含字母+数字）、教师邀请码（选填）。
- **校验**：手机号唯一性；邀请码有效性（未过期、未使用）。
- **输出**：注册成功，自动登录并跳转至对应角色首页。

#### FR-02 用户登录
- **描述**：手机号+密码登录。
- **附加功能**：记住密码、忘记密码（短信重置）。

#### FR-03 管理员预设账户
- **描述**：系统初始化时内置一个管理员账户（账号：admin，密码：系统预设，首次登录强制修改）。

### 3.2 班级管理（教师）

#### FR-04 创建班级
- **描述**：教师创建课程班级。
- **输入**：班级名称（必填）、学期（必填）。小组人数不固定，至少 1 人即可成组，一人成组时互评不适用。
- **输出**：生成班级编号（规则：年份+4位自增，如20250001），班级状态为“进行中”。

#### FR-05 生成班级邀请码
- **描述**：教师为班级生成一次性邀请码。
- **规则**：
  - 有效期24小时，使用后失效。
  - 教师可随时“刷新”邀请码，旧码立即失效，生成新码。
- **输出**：邀请码（6-8位字母数字混合），显示在班级管理页面。

#### FR-06 解散班级
- **描述**：教师结束班级。
- **处理**：
  - 软删除（`is_deleted=1`，`deleted_at=当前时间`）。
  - 班级内所有任务、小组、互评等数据同时软删除。
  - 30天内可恢复（恢复后状态变更为“进行中”，需重新生成邀请码）。
  - 30天后物理删除。

#### FR-07 学生管理
- **描述**：教师查看班级学生列表，可手动移除学生。
- **移除后**：该学生的历史数据保留至学期结束（软删除，仅标记离开），不可再参与当前班级任何新任务。

#### FR-08 分组锁定
- **描述**：教师可锁定分组，锁定后学生无法自建小组或修改分组。

### 3.3 加入班级（学生）

#### FR-09 通过邀请码加入班级
- **描述**：学生输入班级邀请码，验证通过后成为班级成员。
- **校验**：邀请码有效（未过期、未使用、对应班级未解散）。
- **输出**：学生加入班级，状态为“正常”。

### 3.4 分组管理

#### FR-10 教师手动分组
- **描述**：教师为班级指定小组及组长。
- **操作**：选择学生 → 创建小组 → 输入小组名称 → 指定组长。
- **约束**：一人只能属于一个小组；一个班级可有多组。

#### FR-11 学生自建分组
- **描述**：学生（将成为组长）创建小组。
- **创建时设置**：
  - 小组名称
  - 邀请码有效期（默认24小时，单位：天/小时/分钟）
  - 加入模式：**直通**（输入邀请码直接加入）或 **审批**（需组长审批）
- **生成邀请码**：自动生成唯一邀请码。

#### FR-12 加入小组（学生）
- **描述**：学生通过小组邀请码申请加入。
- **流程**：
  - 直通模式：输入邀请码 → 验证通过 → 直接成为成员。
  - 审批模式：输入邀请码 → 提交申请 → 组长审批 → 通过后成为成员。

#### FR-13 小组管理（组长）
- **权限**：
  - 移除成员（无需理由，无申诉）
  - 刷新邀请码（旧码失效，新码生成）
  - 审批入组申请（仅审批模式）
- **约束**：组长不可主动退出小组，只能由教师或管理员移除。

#### FR-14 教师移动成员
- **描述**：教师可将学生从A小组直接移动到B小组。
- **处理**：系统自动执行移出+加入，数据保留。

### 3.5 任务管理

#### FR-15 教师发布总任务
- **描述**：教师为班级创建一项小组作业。
- **输入**：
  - 任务名称
  - 描述
  - 总体截止时间（日期+时间）
  - 是否开启互评（是/否）
  - 若开启：互评时长（小时，默认1小时）
- **输出**：任务状态为“未开始”，到截止时间自动变为“进行中”。

#### FR-16 互评截止时间管理
- **自动计算**：`互评截止时间 = 任务截止时间 + 互评时长`
- **教师可手动修改**：只能延后，不可提前。
- **互评入口关闭**：到达互评截止时间后自动关闭。
- **教师重新开启互评**：可手动延长互评时间（重新设置截止时间）。

#### FR-17 组长拆解子任务
- **描述**：组长将总任务拆解为多个子任务。
- **输入**（每个子任务）：
  - 子任务名称
  - 描述（可选）
  - 质量要求（可选）
  - 截止时间（不能晚于总任务截止时间）
  - 负责人（可指定组员，或留空待认领）
- **操作**：可修改/删除已发布的子任务；可重新指派已被认领的任务（通知原认领人）。

#### FR-18 子任务认领与指派
- **组员**：可自行认领未分配的子任务。
- **组长**：可直接指派子任务给任意组员。

#### FR-19 子任务提交与审批
- **组员提交**：填写完成说明，上传附件（图片/文档/链接）。
- **组长审批**：
  - **通过**：子任务状态变为“已完成”，计入进度。
  - **打回**：子任务状态变为“进行中”，不计入进度，组员需修改后重新提交。
- **注意**：组长审批不是必经环节；无超时自动通过机制。

#### FR-20 逾期标记
- **描述**：子任务超过截止时间未完成 → 自动标记“逾期”（`is_overdue=1`）。
- **逾期后**：仍可提交和审批。

### 3.6 进度可视化

#### FR-21 个人进度
- **计算**：`本人已完成子任务数 / 本人认领的子任务总数`
- **展示**：百分比进度条，实时更新。

#### FR-22 小组进度
- **计算**：`小组已完成子任务数 / 小组总子任务数`
- **展示**：小组看板，所有成员可见。

#### FR-23 教师看板
- **描述**：教师查看班级内所有小组及成员的进度。
- **数据**：每个小组的总体进度、每个成员的子任务完成情况。
- **实时性**：通过 WebSocket 推送，延迟 ≤2秒。

### 3.7 互评机制

#### FR-24 匿名互评
- **开启条件**：教师发布任务时开启互评，且任务已截止、互评未关闭。
- **评分方式**：
  - 教师自定义评分范围（默认为0-100分，整数或一位小数）
  - 可填写评语（文本，可选）
- **匿名性**：
  - 学生端：完全匿名（不显示评价人）
  - 教师端：可见评价人（用于处理申诉）

#### FR-25 未提交互评处理
- 截止后未提交互评的成员：互评分数记为 **NULL**
- 报告中显示“未参与互评”
- 教师不可手动补充评价，但可**重新开启互评**（延长截止时间）让忘记的人补评

### 3.8 评分体系

#### FR-26 教师评分（每次任务）
- **个人评分**：教师对每个学生打分（0-教师自定义上限）。
- **小组评分**：教师对每个小组打分。
- **存储**：`teacher_scores` 表。

#### FR-27 权重设置
- **描述**：教师可为每次任务设置互评与教师评分的权重。
- **示例**：互评占40%，教师评分占60%。系统通过数据库约束确保互评权重与教师评分权重之和为 1（容差范围内）。
- **影响**：系统自动计算该任务每个学生的最终成绩。

#### FR-28 课程总评分
- **自动计算**：教师选择各任务成绩按权重加权汇总。
- **手动打分**：教师直接为每个学生/小组输入课程总分。
- **存储**：`course_scores` 表。

#### FR-29 评分修改重算
- **触发**：教师修改任何评分（任务评分、互评、权重）后，系统自动重新计算相关报告和课程总评分。

### 3.9 申诉流程

#### FR-30 发起申诉
- **可申诉事项**：子任务被误判、互评分数异常、教师评分不公。
- **输入**：申诉理由（必填）、附件（可选，支持图片/文档/链接）。
- **约束**：每个学生对同一任务最多申诉3次。由后端代码通过 COUNT 查询实现。

#### FR-31 处理申诉（教师）
- **操作**：查看申诉内容 → 给出处理结果（同意/驳回）→ 填写反馈。
- **输出**：系统发送站内通知学生结果。
- **后续**：教师可根据申诉结果修改评分或子任务状态。

### 3.10 站内通知

#### FR-32 通知触发
- 子任务即将截止（剩余时间 ≤ 任务时长的10%）
- 组长打回子任务
- 互评开启
- 互评即将截止（剩余时间 ≤ 互评时长的10%）
- 申诉结果通知
- 新成员加入小组
- 子任务被重新指派/取消认领

#### FR-33 通知操作
- 用户可标记单条/全部已读
- 支持批量删除

### 3.11 报告与数据导出

#### FR-34 贡献度报告（每次任务）
- **内容**：
  - 每位成员：子任务完成率、按时完成率、互评平均分（若参与）、教师评分、最终成绩。
  - 小组平均分。
- **格式**：网页展示，支持导出PDF（P1）。

#### FR-35 课程总报告
- **内容**：每位学生/小组的课程最终成绩及历史任务明细。
- **导出**：PDF/Excel（P1）。

### 3.12 数据管理（管理员）

#### FR-36 账户管理
- 查看/搜索所有教师、学生账户。
- 创建教师邀请码（设置有效期）。
- 手动禁用/启用账户。
- 重置密码。

#### FR-37 系统监控（只读）
- 查看所有班级、任务、小组数据（仅查看，不可编辑）。
- 查看操作日志。

### 3.13 操作日志

#### FR-38 日志记录
- **操作类型**：创建、修改、删除。
- **目标**：班级、任务、小组、子任务、成员关系、评分、申诉。
- **记录内容**：操作人ID、时间、IP、UserAgent、变更前后值（JSON）。

### 3.14 定时任务

#### FR-39 后台任务
| 任务               | 频率   | 说明                             |
| ------------------ | ------ | -------------------------------- |
| 子任务提醒         | 每小时 | 计算即将到期子任务，发通知       |
| 互评即将截止提醒   | 每小时 | 互评剩余时间 ≤10%时发通知        |
| 互评自动关闭       | 每小时 | 到达互评截止时间后关闭入口       |
| 软删除数据物理删除 | 每天   | 删除 `deleted_at` 超过30天的记录 |
| 邀请码清理         | 每天   | 删除已过期且未使用的邀请码       |

## 4. 非功能需求

### 4.1 性能
- 页面加载时间 ≤2秒（首屏）
- WebSocket 消息推送延迟 ≤2秒
- 支持并发：单班级同时在线人数 ≥200人

### 4.2 安全性
- 密码加密存储（bcrypt）
- JWT 认证，有效期7天，支持刷新
- 权限中间件：严格校验角色与资源归属
- 防SQL注入：参数化查询
- HTTPS 传输

### 4.3 可用性
- 移动端自适应（响应式布局）
- 关键操作二次确认（解散班级、删除子任务等）
- 表单实时校验，错误提示清晰

### 4.4 可维护性
- 代码模块化，遵循 RESTful API 设计
- 完整的操作日志，便于追溯
- 数据库表包含 `created_at`/`updated_at`

### 4.5 可扩展性
- 预留“分工模板库”表结构
- 预留多语言支持（后期）

## 7. 界面与交互概要

### 7.1 学生端（移动优先）
- **首页**：我的班级列表、待办任务（子任务即将截止、待互评）
- **班级详情**：任务列表、小组看板（进度、成员）
- **小组看板**：任务拆解树、成员进度条、邀请码管理
- **任务详情**：子任务列表、提交入口、审批记录
- **互评页面**：组员列表、评分滑块（0-100）、评语输入
- **申诉页面**：选择申诉类型、填写理由、上传附件

### 7.2 教师端（PC后台 + 移动适配）
- **仪表盘**：班级卡片、进行中任务、待处理申诉数
- **班级管理**：学生列表、分组管理、邀请码生成
- **任务管理**：创建任务、查看各小组进度（看板）、评分入口
- **申诉处理**：列表展示申诉，支持查看详情、处理
- **报告页面**：任务报告、课程总评，支持导出

### 7.3 管理员端（PC后台）
- **账户管理**：教师/学生增删改查，生成教师邀请码
- **班级只读浏览**：查看所有班级数据，不可编辑
- **操作日志查询**：按用户、操作类型、时间筛选

## 8. 验收标准

- 所有 P0 功能按需求实现，无重大 Bug。
- 角色权限正确：学生不能看到其他班级数据，教师只能管理自己班级。
- 实时进度看板在提交、打回后 2 秒内刷新。
- 互评匿名性：学生端看不到评价人，教师端可见。
- 数据软删除后30天内可恢复，30天后物理删除。
- 操作日志记录所有关键动作。
- 提供完整的 API 文档和部署手册。
